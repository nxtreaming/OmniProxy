# v1.0.10-beta.4 - Account modal beta / 账号弹窗 beta

This beta release refines account management interactions and corrects prerelease packaging back to the normal production installer profile. It is intended for user beta testing and will install over the regular production location.
本次 beta 版本优化账号管理交互，并将预发布打包方式修正回正常生产安装包 profile。它面向用户测试，会覆盖常规生产安装位置。

## Beta Notes / Beta 说明
- **Prerelease**: This build is intended for user validation before the stable `v1.0.10` release.
  - **预发布**: 此版本用于正式 `v1.0.10` 前给用户验证。
- **Packaging**: Beta installers use the production profile and production install target, not the Dev profile.
  - **打包方式**: beta 安装包使用生产 profile 和生产安装位置，不使用 Dev profile。

## Changes / 更新内容
- **Delete Account Dialog**: Replace the browser-native delete confirmation with an in-app destructive confirmation dialog that shows account, provider, credential type, and deletion impact.
  - **删除账号弹窗**: 删除确认改为应用内危险操作弹窗，展示账号、厂商、凭据类型和删除影响。
- **Account Modal Motion**: Add a restrained enter/exit transition for adding and editing accounts so the modal no longer appears abruptly.
  - **账号弹窗动效**: 为添加和编辑账号弹窗增加克制的进入/退出动画，避免生硬闪现。
- **Modal Layering**: Improve account modal and confirmation backdrop layering so dialogs stay above the desktop shell and custom title bar.
  - **弹窗层级**: 调整账号弹窗和确认遮罩层级，确保位于桌面外壳和自定义标题栏之上。

## Bug Fixes / 问题修复
- **Release Profile**: Restore prerelease installer builds to the normal production installer behavior.
  - **发布 profile**: 修复预发布安装包打包方式，恢复为正常生产安装包行为。

## Breaking Changes / 不兼容更新
- **Attention**: No backend API or data format breaking changes are expected in this beta release.
  - **必读**: 本 beta 版本预计没有后端接口或数据格式不兼容更新。
