# v1.1.4-beta.3 - model outbound proxy / 模型出站代理

This beta adds model-scoped outbound proxy routing for users who run local tools such as Clash or v2rayN. OmniProxy can now send selected upstream model requests through a local proxy port without requiring a system-wide proxy.

本次 beta 新增按模型选择的出站代理能力，适合正在使用 Clash、v2rayN 等本地代理工具的环境。OmniProxy 可以只让指定上游模型请求走本地代理端口，不需要开启系统全局代理。

## Features / 新特性

- **Model outbound proxy**: Added an outbound proxy setting with support for local HTTP, HTTPS, SOCKS5, and SOCKS5H endpoints, including shorthand inputs such as `10808`, `:10808`, and `mixed:10808`.
  - **模型出站代理**：新增出站代理设置，支持本地 HTTP、HTTPS、SOCKS5、SOCKS5H 端点，并兼容 `10808`、`:10808`、`mixed:10808` 等简写输入。
- **Recommended model routing**: Replaced manual rule text management with grouped model cards. OpenAI/Codex, Claude, Gemini, and provider/model style IDs are selected by default for mainland China networks, while common domestic providers stay direct by default.
  - **推荐模型路由**：不再依赖手动输入规则，改为按模型分组选择。国内网络下默认选中 OpenAI/Codex、Claude、Gemini 和聚合平台常见的 provider/model ID，常见国内模型默认直连。
- **WebSocket proxy support**: Codex WebSocket requests can now use the same model-scoped outbound proxy decision as normal HTTP requests.
  - **WebSocket 代理支持**：Codex WebSocket 请求现在会复用与普通 HTTP 请求一致的按模型出站代理判断。
- **Settings UI polish**: Reworked the Codex WebSocket and model outbound proxy controls into compact switch-style controls, and added selectable model cards for proxy routing.
  - **设置界面优化**：将 Codex WebSocket 和模型出站代理控制优化为更紧凑的开关样式，并新增可选择的模型卡片。

## Bug Fixes / 问题修复

- **Gemini model matching**: Extracted Gemini model names from `/models/{model}:...` request paths so model-scoped outbound proxy rules can match Gemini routes.
  - **Gemini 模型匹配**：从 `/models/{model}:...` 请求路径提取 Gemini 模型名，让按模型出站代理规则可以匹配 Gemini 路由。
- **Wildcard matching**: Fixed wildcard model rules such as `*/*` so provider/model style IDs can be matched correctly.
  - **通配规则匹配**：修复 `*/*` 等通配模型规则，确保聚合平台常见的 provider/model ID 能正确匹配。
- **MiMo console cookie removal**: Removed the Xiaomi MiMo console Cookie and HAR import path from settings and validation code. MiMo account checks no longer store or require the console Cookie.
  - **移除 MiMo 控制台 Cookie**：从设置页和校验代码中移除 Xiaomi MiMo 控制台 Cookie 与 HAR 导入入口，MiMo 账号检查不再保存或要求控制台 Cookie。

## Breaking Changes / 不兼容更新

- **Attention**: No backend API breaking changes are expected in this beta release. Existing saved Xiaomi MiMo console Cookie values are ignored after upgrade.
  - **必读**：本 beta 版本预期没有后端 API 不兼容更新。升级后，已有的 Xiaomi MiMo 控制台 Cookie 配置会被忽略。
