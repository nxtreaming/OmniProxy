# v1.0.5 - Desktop diagnostics and multi-account stability / 桌面诊断与多账号稳定性

This release improves the desktop settings and diagnostics experience, separates development builds from production builds, and fixes Codex multi-account routing stability.
本次版本改进了桌面端设置与诊断体验，隔离开发版与生产版运行环境，并修复 Codex 多账号调度稳定性问题。

##  Features / 新特性
- **About**: Added an application information page with version status, update checks, runtime details, service status, and data directory visibility. @mibgb65-cloud
  - 新增关于应用页面，集中展示版本状态、更新检测、运行环境、本机服务状态和数据目录信息。
- **History**: Increased request history retention and records account validation, quota refresh, and health-check events in request history. @mibgb65-cloud
  - 增加请求历史保留数量，并将账号验证、额度刷新和健康检查记录纳入历史列表。
- **Settings**: Added MiMo credential priority selection and collapsed advanced upstream URL sections by default. @mibgb65-cloud
  - 新增 MiMo 凭据优先级选择，并默认收起高级上游地址配置。
- **Development Build**: Added isolated development runtime profile so `OmniProxy-dev.exe` can run alongside the production executable. @mibgb65-cloud
  - 增加开发版运行隔离配置，使 `OmniProxy-dev.exe` 可以和生产版 exe 同时运行。

##  Bug Fixes / 问题修复
- **Codex**: Fixed multi-account routing where only the last logged-in Codex account worked and other accounts returned 401 (#1) @bigofelephant02
  - 修复了 Codex 多账号场景下只有最后一个登录账号有效、其它账号返回 401 的问题。
- **Settings**: Fixed invalid upstream URLs being saved before validation and leaving the proxy unable to restart. @mibgb65-cloud
  - 修复了无效上游地址先被保存、随后导致代理重启失败的问题。
- **Control API**: Fixed control API port changes not restarting the local control listener. @mibgb65-cloud
  - 修复了控制端口修改后本地控制 API 未切换到新端口的问题。
- **Security**: Restricted control-token retrieval to the desktop app origin. @mibgb65-cloud
  - 限制控制令牌只允许桌面应用来源读取。
- **Settings**: Removed the software update entry from Global Settings and kept update checks on the About page. @mibgb65-cloud
  - 从全局设置页移除软件更新入口，更新检测保留在关于应用页。

  - **必读**: 本版本预计没有不兼容更新。
