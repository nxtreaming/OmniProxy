# v1.1.4-beta.4 - provider outbound proxy fix / 厂商出站代理修复

This beta patch replaces the previous model-rule outbound proxy control with provider-based routing. It fixes Codex model discovery and quota refresh requests, and extends outbound proxy handling to OpenRouter and Zo Computer paths that do not always carry a model name.

本次 beta 补丁将上一版的按模型规则出站代理改为按接入厂商控制，修复 Codex 模型列表与额度刷新请求没有模型名时无法走出站代理的问题，并补齐 OpenRouter 与 Zo Computer 这类不一定携带模型名的接口。

## Bug Fixes / 问题修复

- **Provider-scoped outbound proxy**: Outbound proxy settings now choose providers instead of model patterns. OpenAI/Codex, Anthropic, Gemini, OpenRouter, and Zo Computer are selected by default for mainland China networks; domestic providers remain direct by default.
  - **按厂商出站代理**：出站代理设置改为选择接入厂商，不再依赖模型规则。国内网络下默认选中 OpenAI/Codex、Anthropic、Gemini、OpenRouter 和 Zo Computer，常见国内厂商默认直连。
- **Codex metadata routes**: Codex model-list requests such as `/backend-api/codex/models` and maintenance quota refresh requests now use the OpenAI/Codex provider decision even when the request has no model field.
  - **Codex 查询接口**：`/backend-api/codex/models` 等模型列表请求，以及维护额度刷新请求，即使没有模型字段，也会按 OpenAI/Codex 厂商设置决定是否走出站代理。
- **Token refresh and validation clients**: OAuth token refresh and quota validation now use the same provider-scoped outbound proxy client selection as normal proxy requests.
  - **令牌刷新与额度校验**：OAuth 令牌刷新和额度校验请求现在复用与普通代理请求一致的按厂商出站代理选择。
- **OpenRouter and Zo coverage**: OpenRouter model discovery, OpenRouter desktop chat tests, and Zo Computer model mapping/model list requests now honor the outbound proxy provider selection.
  - **OpenRouter 与 Zo 覆盖**：OpenRouter 模型发现、桌面试聊，以及 Zo Computer 模型映射/模型列表请求都会遵循出站代理厂商选择。

## Compatibility / 兼容性

- **Saved config migration**: Existing `outboundProxyModels` settings are kept for compatibility and migrated to provider selections when older config files are loaded.
  - **配置迁移**：已有 `outboundProxyModels` 配置会保留兼容；读取旧配置时会迁移为按厂商选择。
- **Attention**: No backend API breaking changes are expected in this beta release.
  - **必读**：本 beta 版本预期没有后端 API 不兼容更新。
