# v1.2.5-beta.1 - Codex App account login / Codex App 账号接入

This beta changes one-click Codex App setup from a placeholder API key to a ChatGPT account identity while keeping request scheduling in the OmniProxy backend auth pool.

本次 beta 将 Codex App 一键配置从占位 API Key 改为 ChatGPT 账号身份接入，同时继续由 OmniProxy 后端 auth 池调度实际请求。

## Features / 新特性

- **Codex App account login**: One-click setup imports the current valid Codex account when available, or writes one enabled OpenAI `auth.json` account from the OmniProxy auth pool as the ChatGPT login identity.
  - **Codex App 账号登录**：一键配置会优先导入当前有效的 Codex 账号；否则从 OmniProxy auth 池写入一个已启用的 OpenAI `auth.json` 账号作为 ChatGPT 登录身份。
- **Backend account scheduling**: The client login identity does not pin requests to that account. OmniProxy continues to ignore the incoming account ID for scheduling and injects the auth selected by the backend pool.
  - **后端账号调度**：客户端登录身份不会将请求固定到该账号；OmniProxy 继续在调度时忽略客户端账号 ID，并注入后端 auth 池实际选中的鉴权信息。

## Bug Fixes / 问题修复

- **API login fallback**: Removed the local placeholder `OPENAI_API_KEY` that caused Codex App setup to use API authentication instead of ChatGPT account authentication.
  - **API 登录回退**：移除会让 Codex App 进入 API 鉴权模式的本地占位 `OPENAI_API_KEY`。

## Breaking Changes / 不兼容更新

- **Account required**: Codex App one-click setup now requires at least one enabled OpenAI `auth.json` account. It no longer creates an API-key-only login when the auth pool is empty.
  - **需要账号**：Codex App 一键配置现在至少需要一个已启用的 OpenAI `auth.json` 账号；auth 池为空时不再创建仅含 API Key 的登录信息。
