# v1.2.5-beta.2 - Codex API login fallback / Codex API 登录回退

This beta keeps ChatGPT account login as the preferred Codex App setup and restores automatic local API login when no usable OpenAI `auth.json` account is available.

本次 beta 保留 Codex App 优先使用 ChatGPT 账号接入的行为，并在没有可用 OpenAI `auth.json` 账号时恢复自动本地 API 登录。

## Features / 新特性

- **Automatic login fallback**: Codex setup uses ChatGPT account login when the auth pool has a usable account, otherwise it writes a local placeholder key and sets `forced_login_method = "api"`.
  - **自动登录回退**：auth 池有可用账号时使用 ChatGPT 账号登录；否则写入本地占位 Key，并设置 `forced_login_method = "api"`。
- **Backend scheduling preserved**: The placeholder key is only used between Codex App and the local gateway. OmniProxy still selects and injects the real backend credential for each request.
  - **保持后端调度**：占位 Key 只用于 Codex App 到本地网关的接入；每次请求仍由 OmniProxy 选择并注入真实后端凭据。

## Bug Fixes / 问题修复

- **Empty auth pool setup**: Codex one-click setup no longer stops with an error when no `auth.json` account is available.
  - **空 auth 池配置**：没有可用 `auth.json` 账号时，Codex 一键配置不再报错中止。
