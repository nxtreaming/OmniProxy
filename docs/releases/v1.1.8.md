# v1.1.8 - new-api gateway and update workflow polish / new-api 网关与更新流程优化

This stable release consolidates the validated `v1.1.8` beta line. It adds first-class new-api gateway support, improves the Windows in-app update flow from detection through restart-to-install, adds an opt-in Beta update channel, and fixes dashboard usage totals and Token trend chart rendering.

本正式版汇总已验证的 `v1.1.8` beta 版本线，新增一等 new-api 网关支持，优化 Windows 应用内更新从检测到重启安装的完整流程，加入可选 Beta 更新通道，并修复仪表盘累计用量与 Token 趋势图渲染问题。

## Features / 新特性

- **new-api provider**: Account management now includes a `new-api` provider with a default upstream Base URL of `http://127.0.0.1:3000`.
  - **new-api 厂商**：账号管理新增 `new-api` 厂商，默认上游 Base URL 为 `http://127.0.0.1:3000`。
- **OpenAI / Anthropic / Gemini routing for new-api**: Local routes `/newapi`, `/newapi/anthropic`, and `/newapi/gemini` map to matching new-api-compatible upstream protocols and inject the correct auth headers.
  - **new-api 的 OpenAI / Anthropic / Gemini 路由**：新增本地 `/newapi`、`/newapi/anthropic`、`/newapi/gemini`，按协议转发到 new-api 兼容入口并注入正确鉴权头。
- **new-api quota refresh**: Account validation reads `/api/usage/token/` and stores total, used, available, and unlimited quota state in the account quota view.
  - **new-api 额度刷新**：账号校验会读取 `/api/usage/token/`，并在额度视图记录总额度、已用额度、可用额度和不限额状态。
- **Codex new-api setup**: One-click Codex setup can switch Codex to `http://127.0.0.1:<proxyPort>/newapi` with `gpt-5.5` as the default model.
  - **Codex new-api 一键配置**：一键配置可将 Codex 切换到 `http://127.0.0.1:<proxyPort>/newapi`，默认模型为 `gpt-5.5`。
- **Beta update channel switch**: A settings option controls whether OmniProxy includes GitHub Pre-release/Beta releases in update checks.
  - **Beta 更新通道开关**：设置页选项可控制更新检测是否纳入 GitHub Pre-release/Beta 版本。

## Improvements / 优化

- **Restart-to-install updates**: After a Windows update installer is downloaded and SHA256 verified, OmniProxy can start the installer, exit cleanly, wait for installation, and reopen after completion.
  - **重启完成更新**：Windows 更新安装包下载并通过 SHA256 校验后，OmniProxy 可启动安装器、正常退出、等待安装完成并重新打开。
- **Automatic update download**: Available updates with installer and checksum assets now start downloading in the background and prompt only after the package is ready.
  - **自动下载更新**：当新版本包含安装包和校验文件时，会自动后台下载，并在安装包准备完成后再提示用户。
- **Unified update ready prompt**: Titlebar and About page update actions now use the same no-extra-dialog restart install flow.
  - **统一更新准备完成提示**：标题栏与关于页更新入口现在使用同一套无额外弹窗的重启安装流程。
- **Periodic update checks**: OmniProxy checks once shortly after startup, then checks again every 4 hours while the app is running.
  - **周期更新检查**：OmniProxy 会在启动后短暂延迟检查一次，随后运行期间每 4 小时自动检查一次更新。
- **Update state clarity**: About page and titlebar update UI distinguish Beta updates and show clearer downloading, ready, failed, and installing states.
  - **更新状态更清晰**：关于页和标题栏更新 UI 会区分 Beta 更新，并更清晰展示下载中、已准备好、下载失败和安装器已启动等状态。

## Bug Fixes / 问题修复

- **Dashboard proxy token totals**: The dashboard now reads total proxy requests and tokens from an independent billing summary instead of summing only currently configured accounts.
  - **仪表盘代理 Token 累计**：仪表盘改为读取独立账单汇总，不再只汇总当前账号列表。
- **Deleted accounts no longer remove totals**: Clearing request history or deleting accounts no longer reduces the lifetime proxy token total; only clearing billing summaries resets that statistic.
  - **删除账号不再影响累计用量**：清空请求历史或删除账号不会减少累计代理 Token；只有清空账单汇总会重置该统计。
- **Beta update switch interaction**: The Beta update setting row now has a reliable clickable area and an accessible switch label.
  - **Beta 更新开关交互**：Beta 更新设置项的点击区域已修复，并补充开关可访问标签。
- **Trend chart y-axis alignment**: Token and request trend charts now keep y-axis labels, grid lines, and line/area data on the same visual coordinate system.
  - **趋势图 Y 轴对齐**：Token 与请求次数折线图现在会让 Y 轴刻度、网格线、折线和面积图保持在同一套视觉坐标中。
- **Two-point date labels**: When only two days are recorded, the x-axis now shows only the first and last dates instead of duplicating the first date in the middle.
  - **两点日期标注**：仅有两天记录时，X 轴只显示首尾日期，不再在中间重复显示首日日期。
- **Dark-mode dialogs**: Account deletion and update confirmation dialogs now use clearer dark-mode surfaces.
  - **深色模式弹窗**：账号删除和更新确认弹窗已调整为更清晰的深色表面。

## Compatibility / 兼容性

- **Attention**: Existing configurations continue to load. If `newapiBaseUrl` is missing, OmniProxy falls back to `http://127.0.0.1:3000`.
  - **必读**：已有配置可继续加载。缺少 `newapiBaseUrl` 时，OmniProxy 会回退到 `http://127.0.0.1:3000`。
- **Attention**: Existing providers and local routes are unchanged; new-api is additive.
  - **必读**：已有厂商和本地路由不变，new-api 为新增能力。
- **Attention**: Existing daily billing summaries are used to initialize the new lifetime billing summary when the app starts after upgrading. Historical usage that was already deleted or pruned before this version cannot be reconstructed automatically.
  - **必读**：升级后会使用已有每日账单汇总初始化新的累计账单统计；升级前已经被清空或裁剪掉的历史用量无法自动恢复。
- **Attention**: The Beta update setting is a backward-compatible local configuration field. Existing configs continue to load with Beta checks disabled by default.
  - **必读**：Beta 更新设置是向后兼容的本地配置字段。已有配置会继续加载，并默认关闭 Beta 检测。
- **Attention**: No proxy route or credential format breaking changes are expected in this stable release.
  - **必读**：本正式版预计没有代理路由或凭据格式不兼容更新。
