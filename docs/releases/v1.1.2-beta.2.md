# v1.1.2-beta.2 - refactor cleanup / 重构清理

This beta focuses on reducing duplicated code in the proxy, account management, dashboard, and billing areas. It keeps the existing behavior intact while making provider routing, token retry handling, and frontend views easier to maintain.
本次 beta 聚焦代理、账号管理、仪表盘和账单模块的重复代码清理。在保持现有行为不变的前提下，让 provider 路由、token 重试处理和前端页面更容易维护。

## Features / 新特性

- None / 无。

## Bug Fixes / 问题修复

- **Provider Configuration Checks**: Proxy restart detection now compares the shared provider Base URL fields through one helper, reducing the chance of missing provider-specific changes.
  - **Provider 配置检查**: 代理重启判断现在通过统一 helper 比较 provider Base URL 字段，降低遗漏特定 provider 配置变更的风险。

## Breaking Changes / 不兼容更新

- **Attention**: No backend API or local data format breaking changes are expected in this beta release.
  - **必读**: 本 beta 版本预计没有后端 API 或本地数据格式不兼容更新。

## Refactor & Performance / 重构与优化

- **Provider Metadata**: Centralized proxy Base URL validation, validation Base URL selection, route Base URL selection, and proxy config change checks.
  - **Provider 元数据**: 集中代理 Base URL 校验、额度校验 Base URL 选择、路由 Base URL 选择和代理配置变更判断。
- **Proxy Retry Flow**: Shared HTTP and WebSocket token acquisition, refresh, retry, cooldown, and retry-history handling.
  - **代理重试流程**: 复用 HTTP 和 WebSocket 的 token 获取、刷新、重试、冷却和重试历史记录逻辑。
- **Account Business Logic**: Wails methods and local HTTP handlers now use the same account mutation and validation methods.
  - **账号业务逻辑**: Wails 方法和本地 HTTP handler 现在复用同一组账号变更与校验方法。
- **Token Manager**: Extracted shared usable-token filtering and token mutation helpers.
  - **Token Manager**: 抽取可用 token 过滤和 token 状态变更 helper。
- **Dashboard View**: Split the dashboard screen out of `App.vue` into a dedicated Vue component.
  - **仪表盘页面**: 将 dashboard 从 `App.vue` 拆成独立 Vue 组件。
- **Billing Modules**: Split billing price rules, usage aggregation, and canvas report creation into focused frontend modules.
  - **账单模块**: 将账单价格规则、用量聚合和 canvas 报表创建拆成独立前端模块。
