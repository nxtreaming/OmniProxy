# v1.2.6-beta.1 - Reset-credit popover / 刷新卡悬浮详情

This beta moves Codex quota reset credits out of the bottom of each quota card and into a compact account action with a focused popover.

本次 beta 将 Codex 额度刷新卡从账号卡片底部移到紧凑的账号操作区，并使用悬浮窗集中展示详情。

## Features / 新特性

- **Account-level reset-credit action**: A compact `刷新卡 N` action now appears immediately before `选择 / 已选`, keeping reset credits close to other account controls without occupying a full card row.
  - **账号级刷新卡入口**：紧凑的 `刷新卡 N` 操作现位于 `选择 / 已选` 左侧，让刷新卡与账号操作保持在一起，不再占用卡片底部整行。
- **Reset-credit popover**: Clicking the action opens a floating detail panel with available count, nearest expiry, upstream errors, grant/expiry/redemption history, and confirmed redemption access.
  - **刷新卡悬浮详情**：点击入口后通过悬浮窗展示可用张数、最近到期、上游错误、发放/到期/使用记录及确认使用入口。

## Bug Fixes / 问题修复

- **Stale workspace details**: Switching providers, accounts, or workspaces closes reset-credit details that no longer belong to the visible account.
  - **旧工作区详情残留**：切换厂商、账号或工作区时，会自动关闭不再属于当前账号的刷新卡详情。
- **Compact quota cards**: Removed the previous bottom summary and expanded-history blocks so reset-credit content no longer changes the height of the account grid.
  - **紧凑额度卡片**：移除原有底部概览和展开记录区，刷新卡内容不再改变账号卡片网格高度。

## Breaking Changes / 不兼容更新

- None / 无

## Refactor & Performance / 重构与优化

- **Theme-aware portal**: The popover is mounted inside the application shell, avoiding card clipping while inheriting the current light or dark theme.
  - **主题感知弹层**：悬浮窗挂载到应用外壳内，在避免账号卡片裁剪的同时继承当前深色或浅色主题。
- **Accessible responsive interaction**: Added synchronized expanded state, dialog semantics, visible keyboard focus, reduced-motion handling, and low-height/mobile scrolling.
  - **无障碍响应式交互**：补充同步展开状态、对话框语义、可见键盘焦点、减少动画适配以及低高度/移动端滚动。
