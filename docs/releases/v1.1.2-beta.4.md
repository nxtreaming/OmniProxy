# v1.1.2-beta.4 - help and release notes polish / 使用说明与版本说明优化

This beta focuses on the desktop help and update-information experience. It turns the in-app usage guide into an OmniProxy-specific workflow, improves local route discoverability, and renders GitHub release notes as structured full content instead of a truncated text preview.
本次 beta 聚焦桌面端使用说明和更新信息体验，将应用内使用说明改为贴合 OmniProxy 的项目化工作流，增强本地路由速查，并把 GitHub 版本说明改为结构化完整展示，不再只显示截断预览。

## Features / 新特性

- **Project-Aware Help Guide**: Reworked the usage guide around OmniProxy's real operating model: account pool setup, routing strategy, local proxy startup, quota observation, request history, and realtime logs.
  - **项目化使用说明**：按 OmniProxy 的真实操作链路重做使用说明，覆盖账号池准备、路由策略、本地代理启动、额度观测、请求历史和实时日志。
- **Live Readiness Summary**: Added help-page status cards for proxy state, available accounts, scheduling mode, retry threshold, request history retention, and active requests.
  - **实时接入状态**：使用说明页新增代理状态、可用账号、调度模式、重试阈值、历史保留天数和实时请求状态卡。
- **Local Route Reference**: Added a compact route reference for OpenAI-compatible clients, Codex backend, Codex Chat Completions, Claude router, Gemini, sub2api, and Pi Coding Agent.
  - **本地入口速查**：补充 OpenAI 兼容、Codex backend、Codex Chat Completions、Claude router、Gemini、sub2api 和 Pi Coding Agent 的本地入口说明。

## Bug Fixes / 问题修复

- **Release Notes Display**: The About page now shows the full release body instead of truncating it to a short preview.
  - **版本说明展示**：关于应用页现在完整展示发布说明，不再截断为短预览。
- **Markdown Readability**: Basic Markdown headings, lists, paragraphs, and inline code in release notes are rendered as structured UI without using raw HTML injection.
  - **Markdown 可读性**：版本说明中的基础标题、列表、段落和行内代码会转换为结构化界面展示，并避免使用原始 HTML 注入。

## UX Polish / 体验优化

- **Troubleshooting Path**: Added targeted guidance for missing local requests, invalid credentials, 429 or low quota states, Claude Code model mismatches, and concurrency contention.
  - **排查路径**：新增请求未进入本机代理、账号无效、429 或低额度、Claude Code 模型不符合预期、并发占用等常见问题的排查建议。
- **Responsive Help Layout**: The enhanced help page uses dense desktop grids and collapses cleanly to a single column on narrower layouts.
  - **响应式说明布局**：新的使用说明页在桌面端保持高信息密度，窄屏下会收敛为单列布局。

## Breaking Changes / 不兼容更新

- **Attention**: No backend API or local data format breaking changes are expected in this beta release.
  - **必读**：本 beta 版本预计没有后端 API 或本地数据格式不兼容更新。
