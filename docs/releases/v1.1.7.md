# v1.1.7 - focus automation and desktop workflow polish / 放心刷自动化与桌面工作流优化

This stable release consolidates the validated `v1.1.7` beta line. It adds the "放心刷" automation workflow for CLI-driven AI tasks, expands it with Linux.do browser sessions and local browser profile reuse, improves quota and settings layouts, replaces the Windows tray menu with an OmniProxy-styled popup, and fixes request-summary persistence for high-volume local usage.

本正式版汇总已验证的 `v1.1.7` beta 版本线，新增面向 CLI AI 任务的「放心刷」自动化工作流，并扩展 Linux.do 浏览器会话和本机浏览器 Profile 复用能力；同时优化额度页和设置页布局，替换 Windows 托盘菜单为 OmniProxy 风格弹层，并修复高频本地使用时的请求汇总持久化问题。

## Features / 新特性

- **放心刷 automation**: OmniProxy can watch supported CLI traffic through the local proxy, open a media target while a task is running, pause playback when the task becomes idle, and return focus to the original CLI window.
  - **放心刷自动化**：OmniProxy 可监听经过本地代理的受支持 CLI 请求，在任务运行时打开媒体目标，任务空闲后暂停播放并切回原 CLI 窗口。
- **Launch presets and custom targets**: Added built-in Douyin and Bilibili presets, with support for custom executables, shortcuts, and URLs.
  - **打开预设与自定义目标**：新增抖音和哔哩哔哩预设，并支持自定义 exe、lnk 快捷方式或 URL。
- **Linux.do browser mode**: "放心刷" can open Linux.do in a real browser session instead of a short-video/media workflow, without sending Space to the browser on return.
  - **Linux.do 浏览器模式**：「放心刷」可打开 Linux.do 浏览器会话，不再只面向短视频或媒体应用，并且回切时不会向浏览器发送空格。
- **Browser profile picker**: The desktop app can scan local Edge, Chrome, and Firefox profiles so users can pick an existing logged-in browser profile directly.
  - **浏览器用户资料选择**：桌面端可扫描本机 Edge、Chrome、Firefox 用户资料，方便直接选择已有登录态的浏览器 Profile。

## Improvements / 优化

- **Foreground handling for automation**: Browser and CLI focus switching now preserves maximized window size, only restores minimized windows, and brings Linux.do browser windows to the foreground after launch.
  - **自动化窗口切换优化**：浏览器和 CLI 焦点切换会保留最大化窗口尺寸，只恢复最小化窗口，并在 Linux.do 打开后主动将浏览器窗口切到前台。
- **Session reuse for Linux.do**: Later task rounds reuse the already opened Linux.do browser session instead of launching additional browser windows; closed browser windows are detected and reopened when needed.
  - **Linux.do 会话复用**：后续任务轮次复用已打开的 Linux.do 浏览器会话，不再重复启动浏览器；如果原窗口已关闭，会在需要时重新打开。
- **Quota page spacing**: Account quota cards keep more horizontal room at common desktop widths so subscription and proxy-usage panels no longer feel cramped.
  - **额度页间距**：常见桌面宽度下账号额度卡片保留更多横向空间，避免订阅额度和代理用量面板过于拥挤。
- **Global settings layout**: The settings page keeps a single page flow while sections adapt internally to wide screens, reducing large blank areas around the "放心刷" card.
  - **全局设置布局**：设置页保持单页主流程，各分区在宽屏下内部自适应，减少「放心刷」卡片周围的大块空白。
- **Custom tray menu**: The Windows tray menu now uses an OmniProxy-styled dark rounded popup instead of the native white-framed menu.
  - **自绘托盘菜单**：Windows 托盘菜单改为 OmniProxy 风格暗色圆角弹层，不再使用原生白边菜单。
- **Update hint polish**: The titlebar update hint uses a lightweight non-modal popover and only appears in non-development builds when a real update is detected.
  - **更新提示优化**：标题栏更新提示改为轻量非模态浮层，并且只在非开发构建真实检测到新版本时显示。
- **Release workflow reliability**: Release replacement now removes existing releases for the same tag before publishing replacement assets.
  - **发布流程可靠性**：替换同名 tag 发布时会先清理已有 Release，再发布新的安装包资产。

## Bug Fixes / 问题修复

- **Linux.do opens the intended URL**: The Linux.do preset resolves to `https://linux.do/` instead of passing `preset:linuxdo` to the browser.
  - **Linux.do 正确跳转**：Linux.do 预设会解析为 `https://linux.do/`，不再把 `preset:linuxdo` 误传给浏览器。
- **Media targets preserve window size**: Douyin, Bilibili, and fallback URL launches now use a show mode that avoids resizing existing media or browser windows.
  - **媒体目标保留窗口尺寸**：抖音、哔哩哔哩和备用网页打开时避免主动调整已有媒体或浏览器窗口大小。
- **Request count no longer stops at the detail limit**: Daily request totals are written to an independent SQLite summary table, so the request-detail cap only affects detail rows, not aggregate counts.
  - **请求数不再卡在明细上限**：每日请求总数独立写入 SQLite 汇总表，请求明细上限只影响明细列表，不再影响总请求数统计。
- **Clearing request details keeps daily totals**: "Clear request history" removes detailed request rows while retaining daily request summary and billable token summary data.
  - **清空明细保留每日总计**：「清空请求历史」只删除请求明细，已记录的每日请求汇总和账单 Token 汇总会继续保留。
- **Retained summary migration**: Existing SQLite request rows are backfilled into the new daily summary table on first launch after upgrade.
  - **已有数据迁移**：升级后首次启动会把现有 SQLite 请求记录回填到新的每日汇总表。
- **Quota and settings regressions**: Fixed cramped quota grids, overly stretched settings sections, and update controls that appeared misaligned in the About/titlebar areas.
  - **额度与设置回归修复**：修复额度网格过挤、设置页分区拉伸、关于页和标题栏更新入口对齐异常等问题。

## Compatibility / 兼容性

- **Attention**: No proxy route or credential format breaking changes are expected in this stable release.
  - **必读**：本正式版预计没有代理路由或凭据格式不兼容更新。
- **Attention**: This release adds backward-compatible local configuration fields for "放心刷" and a SQLite daily request summary table. Existing config and request history files continue to load; existing SQLite request rows are migrated into the summary table on first launch after upgrade.
  - **必读**：本版本新增向后兼容的「放心刷」本地配置字段和 SQLite 每日请求汇总表。已有配置和请求历史文件会继续加载；升级后首次启动会把现有 SQLite 请求记录迁移到汇总表。
- **Attention**: Linux.do browser profile launching is implemented for Windows. Default browser mode opens through the system default browser and cannot force a specific profile.
  - **必读**：Linux.do 浏览器 Profile 启动目前面向 Windows。默认浏览器模式会通过系统默认浏览器打开，不能强制指定某个 Profile。
