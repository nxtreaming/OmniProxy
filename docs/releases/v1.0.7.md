# v1.0.7 - Provider expansion and client usage visibility / 厂商扩展与客户端账号占用可视化

This release adds Gemini CLI and OpenCode setup support, expands upstream provider routing, and shows which coding tool is using which account.
本次版本新增 Gemini CLI 与 OpenCode 一键配置，扩展上游厂商路由，并展示不同编程工具正在使用的账号。

## Features / 新特性
- **Providers**: Added Zhipu GLM, MiniMax, Gemini, and custom gateway providers. @mibgb65-cloud
  - 新增 Zhipu GLM、MiniMax、Gemini 和自定义网关厂商。
- **OpenCode**: Added an OpenCode router and one-click local OpenCode configuration. @mibgb65-cloud
  - 新增 OpenCode 路由和本地 OpenCode 一键配置。
- **Gemini CLI**: Added native Gemini API forwarding and one-click Gemini CLI configuration. @mibgb65-cloud
  - 新增原生 Gemini API 转发和 Gemini CLI 一键配置。
- **Client Usage**: Added active request tracking so Codex, Claude Code, OpenCode, Gemini CLI, and other tools can display their current account separately. @mibgb65-cloud
  - 新增活跃请求追踪，Codex、Claude Code、OpenCode、Gemini CLI 等工具可分别显示当前使用账号。
- **History & Logs**: Request history, diagnostics, CSV export, and real-time logs now include the detected coding tool. @mibgb65-cloud
  - 请求历史、诊断详情、CSV 导出和实时日志会记录识别到的编程工具。

## Bug Fixes / 问题修复
- **Quota UI**: Fixed exhausted subscription quota rows showing `undefined%`; they now show `0%`. @mibgb65-cloud
  - 修复订阅额度耗尽后显示 `undefined%` 的问题，现在显示 `0%`。
- **History Migration**: Fixed request history database migration for older rows with missing or NULL optional fields. @mibgb65-cloud
  - 修复旧请求历史数据库中可选字段为空时的迁移和启动问题。
- **Client Headers**: Internal OmniProxy client identification headers are no longer forwarded upstream. @mibgb65-cloud
  - OmniProxy 内部客户端识别请求头不会继续转发给上游。

## Breaking Changes / 不兼容更新
- **Attention**: No breaking changes are expected in this release.
  - **必读**: 本版本预计没有不兼容更新。

## Refactor & Performance / 重构与优化
- **Quota Parsing**: Added Zhipu and MiniMax coding quota parsing, plus Gemini usage token parsing. @mibgb65-cloud
  - 新增 Zhipu 与 MiniMax 编程额度解析，并补充 Gemini usage token 解析。
- **Routing**: Extended model-prefix routing for Anthropic-compatible and OpenAI-compatible clients. @mibgb65-cloud
  - 扩展 Anthropic 兼容和 OpenAI 兼容客户端的模型前缀路由。
