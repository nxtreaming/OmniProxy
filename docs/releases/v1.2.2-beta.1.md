# v1.2.2-beta.1 - Gateway routing and release update hardening / 网关路由与发布更新加固

This beta starts the `v1.2.2` line with a gateway-routing refactor and release/update hardening. Client tools now write stable local OmniProxy entrypoints, while backend provider, credential type, and default model selection move to the Gateway Routing page. The release workflow also attaches unsigned macOS DMG assets to beta and stable releases, and the updater now handles platform checksums and stalled downloads more safely.

本次 beta 开始 `v1.2.2` 版本线，包含网关路由重构和发布/更新流程加固。各类编码 CLI 和客户端现在写入稳定的 OmniProxy 本地入口，后端服务商、凭据类型和默认模型统一在网关路由页面选择。发布流程现在会为 beta 和正式版附加未签名 macOS DMG，更新器也更严格地处理平台校验文件和卡住的下载。

## Features / 新特性

- **Gateway routes**: Added configurable routes for Codex, Claude, OpenAI-compatible clients, and Gemini clients.
  - **网关路由**：新增 Codex、Claude、OpenAI 兼容客户端和 Gemini 客户端的可配置路由。
- **Standalone Gateway Routing page**: Gateway routes now have their own navigation page, with common model preset buttons plus a custom model input for each route.
  - **独立网关路由页面**：网关路由现在作为独立导航页面展示，每条路由提供常用模型快捷按钮和自定义模型输入。
- **Stable client entrypoints**: Codex, Claude Code, Claude Desktop, OpenCode, Pi Coding Agent, DeepSeek-TUI, and Gemini CLI now write stable local gateway URLs.
  - **稳定客户端入口**：Codex、Claude Code、Claude Desktop、OpenCode、Pi Coding Agent、DeepSeek-TUI 和 Gemini CLI 现在写入稳定本地网关地址。
- **Gateway-level provider selection**: Switching between OpenAI, Anthropic, Gemini, DeepSeek, MiMo, sub2api, new-api, AnyRouter, Zo, Prem, or custom gateways no longer requires rewriting client config files.
  - **网关层服务商选择**：切换 OpenAI、Anthropic、Gemini、DeepSeek、MiMo、sub2api、new-api、AnyRouter、Zo、Prem 或自定义网关时，不再需要重写客户端配置文件。
- **macOS DMG release assets**: Beta and stable GitHub Releases now attach the unsigned universal macOS DMG and its SHA256 checksum.
  - **macOS DMG 发布附件**：beta 和正式 GitHub Release 现在都会附加未签名 universal macOS DMG 及其 SHA256 校验文件。

## Breaking Changes / 不兼容更新

- **Deprecated provider-specific setup APIs removed**: Old one-click setup endpoints for Codex sub2api/new-api/AnyRouter/Zo/Prem and Claude provider-specific quick actions were removed.
  - **移除旧服务商专用配置 API**：移除了 Codex sub2api/new-api/AnyRouter/Zo/Prem 以及 Claude 服务商专用快捷配置端点。

## Bug Fixes / 问题修复

- **Checksum matching**: Update checks no longer reuse another platform's `.sha256` file when the selected installer or DMG is missing its own checksum.
  - **校验文件匹配**：当选中的安装包或 DMG 缺少对应校验文件时，不再误用其它平台的 `.sha256` 文件。
- **Download timeout**: Background update downloads now fail cleanly after a long stall instead of remaining in the downloading state indefinitely.
  - **下载超时**：后台更新下载长时间卡住后会明确失败，不再无限停留在下载中。
- **Beta version ordering**: Pre-release comparison now treats numeric suffixes correctly, so `beta.10` sorts after `beta.9`.
  - **Beta 版本排序**：预发布版本比较现在正确处理数字后缀，`beta.10` 会排在 `beta.9` 之后。
- **Windows installer lifecycle**: The NSIS installer now handles auto-update waiting, uninstall-time process shutdown, autostart cleanup, silent uninstall data flags, and shortcut preservation more consistently.
  - **Windows 安装器生命周期**：NSIS 安装器现在更一致地处理自动更新等待、卸载时关闭进程、自启动清理、静默卸载数据参数和快捷方式保留。

## Refactor & Documentation / 重构与文档

- **Settings component split**: Split Global Settings into focused Vue components for maintenance, service and scheduling, task automation, outbound proxy, and provider URLs.
  - **设置页组件拆分**：将全局设置拆分为应用维护、本机服务与调度、放心刷、出站代理和上游地址等独立 Vue 组件。
- **Client setup cleanup**: Removed stale frontend service wrappers and Wails bindings for provider-specific setup actions, leaving stable setup and restore methods.
  - **客户端配置清理**：清理旧的前端服务封装和 Wails 绑定，只保留稳定配置与恢复方法。
- **macOS update copy**: In-app update prompts now describe the unsigned macOS flow as opening a DMG and manually replacing the app in Applications.
  - **macOS 更新文案**：应用内更新提示现在明确说明未签名 macOS 流程是打开 DMG，并手动替换 Applications 中的应用。
- **Docs refresh**: Updated README and help content to explain stable gateway entrypoints and gateway-level backend selection.
  - **文档刷新**：更新 README 与帮助内容，说明稳定网关入口和网关层后端选择方式。
