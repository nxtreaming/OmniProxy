# v1.2.2-beta.3 - Update diagnostics and release checklist / 更新诊断与发布检查清单

This beta continues the `v1.2.2` update hardening line with user-facing diagnostics for the updater and a formal release checklist. It helps validate update state recovery, installer asset cleanup, and release promotion before the stable build.

本次 beta 延续 `v1.2.2` 更新流程加固，新增面向用户的更新诊断信息和正式发布检查清单，用于在正式版前验证更新状态恢复、安装包清理和发布提升流程。

## Features / 新特性

- **Update diagnostics**: About now shows updater state, update directory, `status.json`, `update.log`, installer counts, partial download counts, and recent update logs.
  - **更新诊断**：关于页面现在展示更新状态、更新目录、`status.json`、`update.log`、安装包数量、临时下载文件数量和最近更新日志。
- **Diagnostics copy**: Users can copy update diagnostics for troubleshooting without manually opening temp folders.
  - **诊断复制**：用户可以一键复制更新诊断信息，不需要手动打开临时目录排查。
- **Control API diagnostics**: Added `GET /api/update/diagnostics` for local scripts and the desktop fallback API path.
  - **控制 API 诊断**：新增 `GET /api/update/diagnostics`，供本地脚本和桌面 fallback API 使用。

## Documentation / 文档

- **Release checklist**: Added `docs/release-checklist.md` covering beta validation, Windows install/update/uninstall, unsigned macOS DMG checks, release assets, and stable promotion.
  - **发布检查清单**：新增 `docs/release-checklist.md`，覆盖 beta 验证、Windows 安装/更新/卸载、macOS 未签名 DMG 检查、发布资产和正式版提升流程。
- **README update**: Documented the new diagnostics endpoint and linked the release checklist from the release channel section.
  - **README 更新**：补充新的诊断端点，并在发布通道中加入发布检查清单入口。
