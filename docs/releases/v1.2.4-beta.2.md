# v1.2.4-beta.2 - Forge AI provider / Forge AI 厂商接入

This beta adds Forge AI as a native dual-protocol provider for OpenAI Chat Completions and Anthropic Messages, including account scheduling, model catalog sync, route selection, and fixed local endpoints.

本次 beta 将 Forge AI 作为原生双协议厂商接入 OmniProxy，支持 OpenAI Chat Completions 与 Anthropic Messages，并包含账号调度、模型目录同步、网关路由选择和固定本地入口。

## Features / 新特性

- **Forge AI provider**: Added Forge API Key account management, batch import, health checks, retry scheduling, outbound proxy support, and a configurable upstream Base URL.
  - **Forge AI 厂商**：新增 Forge API Key 账号管理、批量导入、健康检查、失败重试调度、出站代理支持和可配置上游 Base URL。
- **Dual-protocol routing**: Added `/forge/v1` for OpenAI Chat Completions and `/forge/anthropic/v1` for Anthropic Messages, with protocol-specific Bearer and `x-api-key` authentication.
  - **双协议路由**：新增用于 OpenAI Chat Completions 的 `/forge/v1` 和用于 Anthropic Messages 的 `/forge/anthropic/v1`，并按协议注入 Bearer 或 `x-api-key` 鉴权。
- **Gateway integration**: Forge can be selected for OpenAI/OpenCode and Claude Code routes, model routes, fallback chains, and routing strategy presets.
  - **网关集成**：Forge 可用于 OpenAI/OpenCode、Claude Code 路由、模型路由、备用链和路由策略预设。
- **Model catalog sync**: Forge models can be loaded from the authenticated `/v1/models` endpoint and merged into the gateway model list.
  - **模型目录同步**：可通过带鉴权的 `/v1/models` 接口同步 Forge 模型，并临时合并到网关模型列表。

## Bug Fixes / 问题修复

- **Versioned URL joining**: Prevented Forge's versioned upstream Base URL from producing duplicate `/v1/v1` paths for both supported protocols.
  - **版本路径拼接**：避免 Forge 上游 Base URL 已带版本号时，在两种协议下产生重复的 `/v1/v1` 路径。
- **Codex protocol boundary**: Forge is excluded from Codex Responses routes and ignored when a shared model route would otherwise send a Codex Responses request to Forge.
  - **Codex 协议边界**：Forge 不会出现在 Codex Responses 路由中；共享模型路由也不会把 Codex Responses 请求发送到 Forge。
