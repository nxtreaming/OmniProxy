# v1.1.7-beta.4 - Linux.do browser mode and settings layout / Linux.do 浏览器模式与设置页布局

This beta continues the `v1.1.7` line with a focused update to the "放心刷" workflow. It adds a Linux.do browser mode for users who want OmniProxy to open a logged-in browser session while CLI tasks are running, and it fixes the global settings layout on wide desktop windows.

本次 beta 继续完善 `v1.1.7` 版本线，重点优化「放心刷」工作流。新增 Linux.do 浏览器模式，适合在 CLI 任务运行时打开带登录态的浏览器会话，同时修复宽屏桌面下全局设置页布局发散的问题。

## Features / 新特性

- **Linux.do browser mode**: "放心刷" can now open `https://linux.do/` instead of a short-video app.
  - **Linux.do 浏览器模式**：「放心刷」现在可以打开 `https://linux.do/`，不再只面向短视频或媒体应用。
- **Browser and profile selection**: Added browser choices for default browser, Microsoft Edge, Google Chrome, and Firefox, with optional user data directory and Profile fields for browsers that keep logged-in state in profiles.
  - **浏览器与用户资料选择**：新增默认浏览器、Microsoft Edge、Google Chrome、Firefox 选择，并支持填写用户数据目录和 Profile，用已有浏览器资料保留登录态。
- **No pause for Linux.do**: Linux.do mode returns focus to the CLI when enabled, but it does not send Space to pause the browser.
  - **Linux.do 不发送暂停键**：Linux.do 模式仍可在任务结束后切回 CLI，但不会向浏览器发送空格暂停。

## Improvements / 优化

- **Global settings layout**: Constrained the settings content width and placed "放心刷" into the existing desktop grid so the page no longer stretches into uneven wide cards on large windows.
  - **全局设置页布局**：限制设置页内容宽度，并让「放心刷」参与原有桌面分栏，避免宽屏窗口下出现过宽长条和布局不均衡。
- **Configuration normalization**: Added backward-compatible normalization for the new launch mode and browser profile fields.
  - **配置规范化**：新增打开模式和浏览器资料字段的兼容规范化，旧配置会继续使用默认值。

## Compatibility / 兼容性

- **Attention**: Linux.do browser profile launching is implemented for Windows. Default browser mode opens the URL through the system browser and cannot force a specific browser profile.
  - **必读**：Linux.do 浏览器 Profile 启动目前面向 Windows。默认浏览器模式会交给系统默认浏览器打开，不能强制指定某个浏览器 Profile。
- **Attention**: Edge and Chrome use the user data directory plus Profile name, for example `%LOCALAPPDATA%\Microsoft\Edge\User Data` with `Default` or `Profile 1`. Firefox accepts a Profile name or Profile path.
  - **必读**：Edge 和 Chrome 使用用户数据目录加 Profile 名，例如 `%LOCALAPPDATA%\Microsoft\Edge\User Data` 与 `Default` 或 `Profile 1`。Firefox 可填写 Profile 名或 Profile 路径。
