# v1.1.2-beta.3 - dashboard activity calendar / 仪表盘活跃日历

This beta improves the dashboard usage overview with a GitHub-style activity calendar and tighter dashboard navigation behavior.
本次 beta 改进仪表盘用量总览，新增 GitHub 风格活跃日历，并修复仪表盘内跳转到账单页的交互。

## Features / 新特性

- **Request Activity Calendar**: Added a 180-day request heatmap on the dashboard, using daily proxy request counts and token totals.
  - **请求活跃日历**：仪表盘新增最近 180 天请求热力图，基于每日代理请求数和 Token 总量展示活跃度。
- **Longer Daily Usage Retention**: Token daily usage stats are now retained for 365 days so dashboard history can accumulate beyond the previous short window.
  - **更长日统计保留期**：账号每日用量统计保留期扩展到 365 天，便于仪表盘逐步积累更长历史。

## Bug Fixes / 问题修复

- **Billing Navigation**: Dashboard “查看明细 / 查看全部” actions now use an explicit parent handler to switch to the billing page reliably.
  - **账单跳转**：仪表盘“查看明细 / 查看全部”现在通过父组件显式处理，确保能稳定跳转到账单页。
- **Dashboard Layout**: Recent logs now show the latest two records beside the activity calendar, keeping both cards aligned without internal scrollbars.
  - **仪表盘布局**：最近日志仅显示最新两条，与活跃日历等高排列，避免内部滚动条和卡片高度错位。

## Breaking Changes / 不兼容更新

- **Attention**: No backend API or local data format breaking changes are expected in this beta release.
  - **必读**：本 beta 版本预计没有后端 API 或本地数据格式不兼容更新。

## Refactor & Performance / 重构与优化

- **Dashboard Component**: The activity calendar is computed locally from existing daily usage rows and does not add a new API endpoint.
  - **仪表盘组件**：活跃日历直接复用现有每日用量数据计算，不新增后端接口。
