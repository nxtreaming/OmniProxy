# v1.0.12 - Account selection and MiMo Claude context / 账号选择与 MiMo Claude 上下文

This stable release consolidates the `v1.0.12` beta series. It adds selected-account scheduling for controlled routing, fixes the right-side desktop resize edge reported in #3, and updates the Xiaomi MiMo Claude Code quickstart so MiMo V2.5 Pro uses the 1M-context route by default while keeping the regular Pro model available.
本正式版整合 `v1.0.12` beta 系列改动，新增账号选择调度用于控制路由范围，修复 #3 反馈的桌面窗口右侧边框无法拖动问题，并更新 Xiaomi MiMo 的 Claude Code 一键配置：MiMo V2.5 Pro 默认使用 1M 上下文路由，同时保留普通 Pro 模型选项。

## Features / 新特性

- **Selected Account Scheduling (#4)**: Providers rotate all usable accounts by default, and switch to rotating only selected accounts once one or more accounts are selected.
  - **账号选择调度 (#4)**: 厂商默认轮换全部可用账号；当选择一个或多个账号后，仅在已选账号内轮换。
- **Multi-account Selection (#4)**: Selecting one account behaves like the old only-use mode, while selecting multiple accounts allows routing through a controlled subset without disabling other accounts.
  - **多账号选择 (#4)**: 选择单个账号等价于旧的仅使用模式；选择多个账号时，可在指定账号子集内轮换，且不会停用其他账号。
- **Selection API**: Added `PUT /api/tokens/{id}/selected` and matching Wails bindings for desktop selection state changes.
  - **选择接口**: 新增 `PUT /api/tokens/{id}/selected` 及对应 Wails 绑定，用于桌面端切换选择状态。
- **Coexisting Dev Build**: Added `scripts/build-dev.ps1` to build `OmniProxy-Dev.exe` with the dev profile, separate data directory, and separate local ports.
  - **共存 Dev 构建**: 新增 `scripts/build-dev.ps1`，可构建使用 dev profile、独立数据目录和独立本地端口的 `OmniProxy-Dev.exe`。

## Bug Fixes / 问题修复

- **Right Window Resize Edge (#3)**: Added a Wails resize edge on the right side of the custom title bar shell so the desktop window can be resized from the right border again.
  - **右侧窗口缩放边缘 (#3)**: 在自定义标题栏外壳右侧补充 Wails 缩放边缘，恢复从右边框调整窗口大小的能力。
- **MiMo Pro Context**: Fixed the Claude Code MiMo preset not enabling the `[1m]` context variant for the Pro model.
  - **MiMo Pro 上下文**: 修复 Claude Code MiMo 预设未为 Pro 模型启用 `[1m]` 上下文变体的问题。
- **Selection Persistence Semantics**: Selection no longer mutates other accounts' disabled state, so clearing a selection does not accidentally re-enable manually disabled accounts.
  - **选择状态语义**: 选择调度不再改写其他账号的停用状态，清空选择时不会误启用手动停用的账号。

## Breaking Changes / 不兼容更新

- **Attention**: No backend API or data format breaking changes are expected in this release. The previous `/exclusive` API remains available as a compatibility wrapper over the new selection model.
  - **必读**: 本版本预计没有后端接口或数据格式不兼容更新。旧 `/exclusive` API 仍作为新选择模型的兼容包装保留。

## Refactor & Performance / 重构与优化

- **Quota Card Actions**: Quota cards now show account information first, then compact `选择 / 已选` and `刷新` actions using consistent status-sized controls.
  - **额度卡片操作**: 额度卡片先展示账号信息，再展示紧凑的 `选择 / 已选` 与 `刷新` 操作，并统一为状态标签大小。
- **Selected State Feedback**: The selected action now uses a persistent highlighted background and a distinct selected icon so selected accounts remain visually obvious.
  - **选中态反馈**: 已选操作使用持续高亮背景和独立选中图标，让账号是否参与选择集合更清晰。
- **Account Management Controls**: Enable and disable controls now live in Account Management, separating account availability from quota-page routing selection.
  - **账号管理控制**: 启用和停用操作归位到账号管理页，将账号可用性与额度页调度选择分离。
- **MiMo Claude Quickstart**: Claude Code one-click MiMo configuration now writes `ANTHROPIC_MODEL` and `ANTHROPIC_DEFAULT_OPUS_MODEL` to `mimo-v2.5-pro[1m]`, while keeping `mimo-v2.5-pro` as `ANTHROPIC_CUSTOM_MODEL_OPTION`.
  - **MiMo Claude 一键配置**: Claude Code 的 MiMo 一键配置现在会将 `ANTHROPIC_MODEL` 和 `ANTHROPIC_DEFAULT_OPUS_MODEL` 写为 `mimo-v2.5-pro[1m]`，同时将 `mimo-v2.5-pro` 保留为 `ANTHROPIC_CUSTOM_MODEL_OPTION`。
- **README Refresh**: Updated Chinese and English README content for supported providers, control APIs, Dev builds, and the current selection workflow.
  - **README 更新**: 更新中英文 README，补充当前支持厂商、控制 API、Dev 构建和账号选择调度流程。
