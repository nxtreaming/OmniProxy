# v1.2.0-beta.3 - frontend bundle split and UI loading polish / 前端拆包与界面加载优化

This beta focuses on reducing the initial frontend bundle by moving page and modal styles into lazy-loaded component chunks. The application functionality remains aligned with `v1.2.0-beta.2`, while the desktop UI should load less global CSS and less initial JavaScript before secondary pages or dialogs are opened.

本次 beta 聚焦前端首包优化，将页面和弹窗样式迁移到按需加载的组件 chunk 中。应用功能与 `v1.2.0-beta.2` 保持一致，但桌面 UI 在打开次级页面或弹窗前需要加载的全局 CSS 和初始 JavaScript 更少。

## Changes / 变更

- **Frontend bundle split**: Lazy-loaded dashboard, settings, history, logs, about, quickstart, help, billing, OpenRouter chat, quota, token, and modal-specific styles into component CSS chunks.
  - **前端拆包**：将仪表盘、设置、历史、日志、关于、快速开始、帮助、账单、OpenRouter 聊天、额度、账号和弹窗相关样式拆入组件 CSS chunk。
- **Dialog loading optimization**: Converted token editor and batch import dialogs to async components so their JavaScript and CSS no longer sit in the initial bundle.
  - **弹窗加载优化**：将账号编辑和批量导入弹窗改为异步组件，使其 JavaScript 和 CSS 不再进入初始包。
- **Main bundle reduction**: Reduced the built main CSS from about `322.36 kB` to about `269.09 kB`, and reduced the built main JS from about `283.55 kB` to about `255.52 kB` during local verification.
  - **主包体积下降**：本地验证中，构建后的主 CSS 由约 `322.36 kB` 降至约 `269.09 kB`，主 JS 由约 `283.55 kB` 降至约 `255.52 kB`。
- **Scoped UI risk control**: Kept shared modal, GeminiSelect, diagnostic drawer, and other cross-component base styles global where they are still reused.
  - **控制样式迁移风险**：对仍被复用的通用弹窗、GeminiSelect、诊断抽屉等基础样式继续保留在全局层。
