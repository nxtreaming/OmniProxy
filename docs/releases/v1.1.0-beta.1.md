# v1.1.0-beta.1 - Pi Coding Agent beta / Pi Coding Agent 接入 beta

This beta release adds Pi Coding Agent one-click configuration and refines Pi routing so the client can use a single OmniProxy OpenAI-compatible entry while OmniProxy dispatches requests by model.
本次 beta 版本新增 Pi Coding Agent 一键配置，并优化 Pi 路由：客户端只需要使用一个 OmniProxy OpenAI 兼容入口，OmniProxy 会根据模型自动分流到对应上游。

##  Features / 新特性

- **Pi Coding Agent**: Added one-click local Pi provider configuration for OmniProxy. @mibgb65-cloud
  - 新增 Pi Coding Agent 本地 provider 一键配置，指向 OmniProxy 本机代理。
- **Pi Router**: Added model-based routing for Pi requests through a single `/pi-router/v1` OpenAI-compatible entry. @mibgb65-cloud
  - 新增 Pi 单入口模型分流，Pi 请求统一走 `/pi-router/v1`，由 OmniProxy 根据 `model` 选择上游。

##  Bug Fixes / 问题修复

- **Pi MiMo Routing**: Fixed Pi advertising the `mimo-v2.5-pro[1m]` model on the OpenAI-compatible route, which could cause Xiaomi MiMo upstream `400` responses and local account cooldowns. @mibgb65-cloud
  - 修复 Pi 在 OpenAI 兼容入口暴露 `mimo-v2.5-pro[1m]`，可能导致 Xiaomi MiMo 上游返回 `400` 并触发本地账号冷却的问题。
- **Pi OpenAI Routing**: Fixed Pi GPT/OpenAI model requests accidentally using Codex `auth.json` credentials instead of OpenAI API keys. @mibgb65-cloud
  - 修复 Pi 的 GPT/OpenAI 模型请求可能误用 Codex `auth.json` 凭据，而不是 OpenAI API Key 的问题。

## ⚠️ Breaking Changes / 不兼容更新

- **Attention**: No breaking changes are expected in this beta release. Re-run Pi one-click configuration to rewrite the local `models.json` into the new single-provider layout.
  - **必读**: 本 beta 版本预计没有不兼容更新。请重新执行 Pi 一键配置，将本机 `models.json` 写入新的单 provider 结构。

##  Refactor & Performance / 重构与优化

- **Pi Configuration**: Removed separate OmniProxy Pi provider entries for Anthropic, Gemini, OpenRouter, and Custom gateways in favor of one model-routed provider. @mibgb65-cloud
  - 优化 Pi 配置，移除独立的 Anthropic、Gemini、OpenRouter、自定义网关 provider，改为单 provider 模型分流。
