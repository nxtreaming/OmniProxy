# v1.0.11 - OpenRouter and CLIProxyAPI support / OpenRouter 与 CLIProxyAPI 支持

This stable release adds OpenRouter as a first-class provider and improves local desktop workflows around model discovery, direct chat testing, quota visibility, OpenCode configuration, and Codex CLIProxyAPI auth JSON import.
本正式版新增 OpenRouter 独立厂商支持，并围绕模型发现、直接对话测试、额度展示、OpenCode 配置和 Codex CLIProxyAPI auth JSON 导入优化本地桌面工作流。

## Features / 新特性

- **Codex CLIProxyAPI**: Support importing Codex CLIProxyAPI auth JSON credentials and refreshing compatible Codex credential state.
  - **Codex CLIProxyAPI**: 支持导入 Codex CLIProxyAPI auth JSON 凭据，并刷新兼容的 Codex 凭据状态。
- **OpenRouter Provider**: Add OpenRouter as a first-class provider with its own base URL, account type, proxy route, validation path, quota handling, and Wails bindings.
  - **OpenRouter 提供商**: 新增 OpenRouter 独立厂商入口，支持独立 Base URL、账号类型、代理路由、验证路径、额度处理和 Wails 绑定。
- **OpenRouter Models**: Fetch and cache OpenRouter model metadata, including model IDs, display names, context length, and pricing metadata.
  - **OpenRouter 模型**: 支持拉取并缓存 OpenRouter 模型元数据，包括模型 ID、展示名称、上下文长度和价格信息。
- **OpenRouter Chat Playground**: Add a dedicated OpenRouter chat page for selecting a model and testing conversations directly inside OmniProxy.
  - **OpenRouter 对话体验页**: 新增独立 OpenRouter 对话页面，可选择模型并直接在 OmniProxy 内测试对话。
- **Free Model Filtering**: Add quick filtering for free OpenRouter models and paginated model browsing in account management.
  - **免费模型筛选**: 支持一键筛选 OpenRouter 免费模型，并在账号管理中分页浏览模型列表。
- **OpenCode Integration**: OpenCode configuration can now include an OmniProxy OpenRouter provider and model list when available.
  - **OpenCode 集成**: OpenCode 配置可写入 OmniProxy OpenRouter provider，并在可用时同步模型列表。

## Bug Fixes / 问题修复

- **OpenRouter Quota Refresh**: Fixed refresh feedback when OpenRouter `/key` returns usage without `limit` or `limit_remaining`.
  - **OpenRouter 额度刷新**: 修复 OpenRouter `/key` 只返回 usage、没有 `limit` 或 `limit_remaining` 时刷新后看起来无变化的问题。
- **Refresh Accounting**: Batch quota refresh now counts failed validation responses correctly instead of treating every HTTP response as success.
  - **刷新结果统计**: 批量刷新额度会正确统计验证失败结果，不再把所有 HTTP 返回都当成成功。
- **Model Selection UX**: Fixed OpenRouter chat model selection and model-list loading states for large model lists.
  - **模型选择体验**: 修复 OpenRouter 对话页在大量模型场景下的选择与加载状态体验。

## Breaking Changes / 不兼容更新

- **Attention**: No backend API or data format breaking changes are expected in this release.
  - **必读**: 本版本预计没有后端接口或数据格式不兼容更新。

## Refactor & Performance / 重构与优化

- **Quota Visibility**: Show OpenRouter API Key quota on both the quota page and the OpenRouter chat page, including unlimited keys without configured spend limits.
  - **额度可见性**: OpenRouter API Key 额度会同时显示在额度页和 OpenRouter 对话页；未设置消费上限的 Key 会显示为“不限制”。
- **Chat Experience**: Save OpenRouter conversations locally per model and improve generation feedback, local streaming output, input focus styling, send-button contrast, and message layout stability.
  - **对话体验**: OpenRouter 对话按模型本地保存，并优化生成反馈、本地流式输出、输入框焦点样式、发送按钮对比度和消息布局稳定性。
- **Dashboard Density**: Dashboard quota and daily usage panels now use compact paging and summary trends so long account lists and large charts do not stretch the home page.
  - **仪表盘密度优化**: 首页额度和分天用量面板改为紧凑分页与摘要趋势，避免账号列表和大图表拉长首页。
