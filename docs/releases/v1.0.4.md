# OmniProxy v1.0.4

第五个公开版本，聚焦本地控制接口安全、Token 加密保存、Codex 长时间使用时的 Token 自动刷新，以及凭据迁移和验证覆盖。

## 下载

- `OmniProxy-Setup-v1.0.4-windows-amd64.exe`：Windows 安装包，推荐下载。
- `OmniProxy-Setup-v1.0.4-windows-amd64.exe.sha256`：安装包 SHA256 校验文件。

## 亮点

- 增强本地控制 API 访问保护，降低其他本机进程误调用敏感管理接口的风险。
- Token 本地存储改为加密保存，减少配置文件泄露真实凭据的风险。
- Codex 请求转发前会检查 Token 状态；遇到 Token 过期时会自动刷新并重试，减少长时间使用后的认证失败。
- 新增凭据导出流程，方便备份和迁移账号配置，同时延续敏感信息保护策略。
- 修复旧历史记录导入后缺少导入 ID 的问题，提升历史迁移和排查时的可追踪性。
- 前端测试纳入 CI，并补充 Codex 导入和历史迁移相关测试覆盖。
- 更新安全说明与验证记录，明确本地监听和凭据保护边界。

## 升级说明

- 从旧版本升级后，已有 Token 会按新版逻辑迁移为加密保存。
- 正在使用 Codex 的场景下，Token 过期会自动刷新后重试，一般不需要手动重启应用。
- 该版本安装包由 GitHub Actions 构建发布，版本号注入为 `v1.0.4`。

## 安全提示

OmniProxy 默认面向本地个人开发场景，只监听 `127.0.0.1`。请不要把真实凭据提交到 GitHub。
