# v1.1.4 - onboarding, client references, and outbound proxy / 首次引导、客户端速查与出站代理

This stable release consolidates the `v1.1.4` beta series. It adds first-run onboarding, a broader third-party client endpoint reference, Zo Claude routing for Claude clients, and provider-scoped outbound proxy support for local proxy tools such as Clash or v2rayN.

本正式版整合 `v1.1.4` beta 系列改动，新增首次使用引导、更完整的第三方客户端接口速查、Claude 客户端的 Zo Claude 路由，并补充适配 Clash、v2rayN 等本地代理工具的按厂商出站代理能力。

## Features / 新特性

- **First-Run Guide**: Added a one-time onboarding dialog that walks new users through adding accounts, starting the local proxy, and opening one-click client setup.
  - **首次使用引导**：新增首次打开时自动显示一次的引导框，提示添加账号、启动本地代理和进入一键配置。
- **Client Endpoint Reference**: Added a complete third-party client endpoint reference in the help page, covering OpenAI-compatible, Anthropic-compatible, vendor-specific, and coding-tool routes with copy actions.
  - **客户端接口速查**：在使用说明中新增第三方客户端接口清单，覆盖 OpenAI 兼容、Anthropic 兼容、厂商直连和编程工具路由，并支持复制配置值。
- **Zo Claude Routing**: Added Zo Claude Opus 4.7 and Sonnet 4.6 options to Claude Code and Claude Desktop model configuration.
  - **Zo Claude 路由**：Claude Code 与 Claude Desktop 模型配置新增 Zo Claude Opus 4.7 和 Sonnet 4.6 选项。
- **Provider outbound proxy**: Added outbound proxy settings for local HTTP, HTTPS, SOCKS5, and SOCKS5H endpoints, with provider selection for OpenAI/Codex, Anthropic, Gemini, OpenRouter, and Zo Computer.
  - **按厂商出站代理**：新增出站代理设置，支持本地 HTTP、HTTPS、SOCKS5、SOCKS5H 端点，并可按 OpenAI/Codex、Anthropic、Gemini、OpenRouter 和 Zo Computer 等接入厂商选择是否走代理。
- **WebSocket proxy support**: Codex WebSocket requests can now use the same outbound proxy decision as normal HTTP requests.
  - **WebSocket 代理支持**：Codex WebSocket 请求现在会复用与普通 HTTP 请求一致的出站代理判断。
- **Settings and billing polish**: Reworked related settings controls into compact switch-style sections and added animated billing preview transitions.
  - **设置与账单体验优化**：将相关设置控制优化为更紧凑的开关区域，并为模拟账单预览加入弹层动效。

## Bug Fixes / 问题修复

- **Codex metadata routes**: Codex model-list requests such as `/backend-api/codex/models` and maintenance quota refresh requests now use the OpenAI/Codex provider proxy decision even when the request has no model field.
  - **Codex 查询接口**：`/backend-api/codex/models` 等模型列表请求，以及维护额度刷新请求，即使没有模型字段，也会按 OpenAI/Codex 厂商设置决定是否走出站代理。
- **Token refresh and validation clients**: OAuth token refresh and quota validation now use the same provider-scoped outbound proxy client selection as normal proxy requests.
  - **令牌刷新与额度校验**：OAuth 令牌刷新和额度校验请求现在复用与普通代理请求一致的按厂商出站代理选择。
- **OpenRouter and Zo coverage**: OpenRouter model discovery, OpenRouter desktop chat tests, and Zo Computer model mapping/model list requests now honor the outbound proxy provider selection.
  - **OpenRouter 与 Zo 覆盖**：OpenRouter 模型发现、桌面试聊，以及 Zo Computer 模型映射/模型列表请求都会遵循出站代理厂商选择。
- **Claude Desktop Zo Gateway**: Fixed Claude Desktop gateway requests for Zo Claude models so they route through the Zo provider instead of the default Anthropic provider.
  - **Claude Desktop Zo 网关**：修复 Claude Desktop 使用 Zo Claude 模型时会落到默认 Anthropic provider 的问题，确保请求进入 Zo provider。
- **Request detail and billing preview UI**: Fixed request-history drawer bounds and dark-mode style leakage in light billing preview templates.
  - **请求详情与账单预览界面**：修复请求历史详情抽屉边界，并修复深色模式全局样式污染浅色账单模板的问题。
- **MiMo console cookie removal**: Removed the Xiaomi MiMo console Cookie and HAR import path from settings and validation code.
  - **移除 MiMo 控制台 Cookie**：从设置页和校验代码中移除 Xiaomi MiMo 控制台 Cookie 与 HAR 导入入口。

## Compatibility / 兼容性

- **Saved config migration**: Existing `outboundProxyModels` settings are kept for compatibility and migrated to provider selections when older config files are loaded.
  - **配置迁移**：已有 `outboundProxyModels` 配置会保留兼容；读取旧配置时会迁移为按厂商选择。
- **Attention**: No backend API or local data format breaking changes are expected in this stable release. Existing saved Xiaomi MiMo console Cookie values are ignored after upgrade.
  - **必读**：本正式版预计没有后端 API 或本地数据格式不兼容更新。升级后，已有的 Xiaomi MiMo 控制台 Cookie 配置会被忽略。
