# v1.1.8-beta.5 - token trend chart alignment / Token 趋势图对齐修复

This beta continues the `v1.1.8` validation line with focused fixes for the Token trend chart. The y-axis scale now stays aligned with the SVG plot area, the chart is easier to read in half-width cards, and short two-day ranges no longer show a duplicated middle date.

本次 beta 继续验证 `v1.1.8` 版本线，集中修复 Token 趋势折线图体验。Y 轴刻度现在会与 SVG 绘图区保持一致坐标，半宽卡片里的图表更易读，两天数据范围也不再显示重复的中间日期。

## Fixes / 修复

- **Trend chart y-axis alignment**: Token and request trend charts now keep y-axis labels, grid lines, and line/area data on the same visual coordinate system.
  - **趋势图 Y 轴对齐**：Token 与请求次数折线图现在会让 Y 轴刻度、网格线、折线和面积图保持在同一套视觉坐标中。
- **SVG aspect-ratio handling**: Trend chart SVGs no longer introduce internal vertical padding from default aspect-ratio preservation.
  - **SVG 宽高比处理**：趋势图 SVG 不再因为默认保持宽高比而产生内部上下留白。
- **Two-point date labels**: When only two days are recorded, the x-axis now shows only the first and last dates instead of duplicating the first date in the middle.
  - **两点日期标注**：仅有两天记录时，X 轴只显示首尾日期，不再在中间重复显示首日日期。

## Improvements / 优化

- **Chart readability**: The trend chart plot area is slightly taller, with clearer y-axis typography and a wider scale column.
  - **图表可读性**：趋势图绘图区略微增高，Y 轴字体更清晰，刻度列也留出更多宽度。

## Compatibility / 兼容性

- **Attention**: This release changes only frontend chart rendering. Local data, account configuration, proxy routes, and API behavior are unchanged.
  - **必读**：此版本仅调整前端图表渲染，本地数据、账号配置、代理路由和 API 行为均不变。
