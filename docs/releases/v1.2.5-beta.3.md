# v1.2.5-beta.3 - Codex browser login and quota reset credits / Codex 浏览器登录与额度刷新卡

This beta adds browser-based Codex account login and surfaces Codex quota reset credits, including their history, expiry, and confirmed redemption.

本次 beta 新增 Codex 浏览器账号登录，并展示 Codex 额度刷新卡的数量、记录、到期时间与确认使用入口。

## Features / 新特性

- **Codex browser login**: Account Management now opens the OpenAI authorization page, completes a PKCE flow through a loopback-only callback, and automatically adds or updates the matching OpenAI `auth.json` account.
  - **Codex 浏览器登录**：账号管理页现可打开 OpenAI 授权页，通过仅本机可访问的回调完成 PKCE 流程，并自动新增或更新对应 OpenAI `auth.json` 账号。
- **Quota reset credit overview**: Codex quota cards show the available count and nearest expiry, with a detail drawer for grant, expiry, redemption time, and status history.
  - **额度刷新卡概览**：Codex 额度卡片展示可用数量和最近到期时间，详情抽屉保留发放、到期、使用时间与状态记录。
- **Confirmed redemption**: Consuming a reset credit requires explicit confirmation, uses an idempotent redemption request ID, and refreshes the five-hour quota and credit details afterwards.
  - **确认后使用**：消耗刷新卡前需明确确认，请求使用幂等兑换 ID，成功后自动刷新 5 小时额度和刷新卡详情。

## Refactor & Performance / 重构与优化

- **Reset-credit detail caching**: Detailed credit records are cached briefly and invalidated immediately after redemption, avoiding an extra upstream request on every quota refresh.
  - **刷新卡详情缓存**：详细记录会短时缓存，并在使用后立即失效，避免每次额度刷新都额外请求上游。
