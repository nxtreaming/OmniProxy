# v1.2.1-beta.3 - Codex estimate model and scheduler stability / Codex 估算模型与调度稳定性

This beta follows `v1.2.1-beta.2` with two focused corrections. Codex weekly USD quota estimates now use the GPT-5.5 price rule by default, and balanced account scheduling is deterministic when multiple accounts share the same creation timestamp.

本次 beta 延续 `v1.2.1-beta.2`，集中修正两个问题：Codex 周额度美元预估默认改用 GPT-5.5 价格规则；当多个账号创建时间戳相同时，平衡调度现在具备确定性的兜底顺序。

##  Features / 新特性

- **Codex quota estimate**: Updated the weekly USD quota estimate to use the local `OpenAI GPT-5.5` price rule by default.
  - **Codex 额度预估**：周额度美元预估默认改用本地 `OpenAI GPT-5.5` 价格规则。

##  Bug Fixes / 问题修复

- **Balanced scheduler**: Fixed nondeterministic account selection when balanced scheduling compares accounts with identical `CreatedAt` timestamps.
  - **平衡调度**：修复账号 `CreatedAt` 时间戳相同时，平衡调度选择结果不稳定的问题。
- **CI stability**: Covered the equal-timestamp scheduling case that could fail on Windows runners with coarse timer precision.
  - **CI 稳定性**：补充同时间戳调度场景覆盖，避免 Windows runner 时间精度较粗时触发偶发失败。

## ⚠️ Breaking Changes / 不兼容更新

- None / 无

##  Refactor & Performance / 重构与优化

- **Scheduler tie-breaker**: Added a stable final tie-breaker that preserves older account-pool order after all usage and quota metrics are equal.
  - **调度兜底排序**：在所有用量与额度指标都相同时，新增稳定兜底排序，优先保留账号池中更早加入的账号。
