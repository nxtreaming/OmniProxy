# v1.1.2 - Codex compatibility and usage insights / Codex 兼容与用量洞察

This stable release consolidates the `v1.1.2` beta series. It adds Codex Chat Completions compatibility, simplifies several proxy/frontend paths, improves dashboard observability, expands usage trend analysis, and refreshes the in-app help and release notes experience.
本正式版整合 `v1.1.2` beta 系列改动，新增 Codex Chat Completions 兼容入口，清理多处代理与前端逻辑，增强仪表盘可观测性，补充完整用量趋势分析，并优化应用内使用说明与版本说明体验。

## Features / 新特性

- **Codex Chat Completions Route**: Added `/codex/v1/chat/completions`, allowing OpenAI Chat Completions compatible clients to use Codex `auth.json` accounts through OmniProxy.
  - **Codex Chat Completions 入口**：新增 `/codex/v1/chat/completions`，OpenAI Chat Completions 兼容客户端可以通过 OmniProxy 使用 Codex `auth.json` 账号。
- **Codex Responses Conversion**: Chat messages, system/developer instructions, max token fields, reasoning effort, tools, and function calling metadata are converted into Codex Responses requests.
  - **Codex Responses 转换**：将聊天消息、system/developer 指令、最大 Token、reasoning effort、tools 与 function calling 元数据转换为 Codex Responses 请求。
- **Chat Response Compatibility**: Codex Responses JSON and streaming SSE events are converted back into `chat.completion` and `chat.completion.chunk` responses.
  - **Chat 响应兼容**：Codex Responses 的 JSON 与流式 SSE 事件会转换回 `chat.completion` 与 `chat.completion.chunk` 响应。
- **Usage Trends Page**: Added a dedicated “用量趋势” page that shows all recorded daily token usage and request counts as trend charts, with totals, peak day, recent comparison, and a daily detail table.
  - **用量趋势页面**：新增独立的“用量趋势”页面，以趋势图展示全部已记录天数的 Token 用量与请求次数，并提供总量、峰值日、近期对比和每日明细表。

## UX Polish / 体验优化

- **Dashboard Activity Calendar**: Added a GitHub-style daily activity calendar to make request activity and token volume easier to scan over time.
  - **仪表盘活跃日历**：新增类似 GitHub 的分天活跃日历，便于按时间浏览请求活跃度和 Token 用量。
- **Dashboard Trend Entry**: The dashboard daily usage panel now links directly to the full usage trends page while keeping billing details as a separate action.
  - **趋势入口优化**：仪表盘分天用量区域新增完整趋势入口，同时保留账单明细入口。
- **Help Page Refresh**: Rebuilt the usage guide around the project’s real workflow, including readiness checks, local routes, account types, client setup, troubleshooting, and security reminders.
  - **使用说明重构**：围绕项目实际工作流重构使用说明，覆盖就绪检查、本地入口、账号类型、客户端配置、排障和安全提醒。
- **Release Notes Layout**: Release notes in the About page now render as a full-width panel with a wider reading layout, so longer bilingual notes no longer feel cramped.
  - **版本说明布局优化**：关于应用里的版本说明改为满宽面板和更宽松的阅读排版，较长的双语说明不再拥挤。

## Bug Fixes / 问题修复

- **Codex Model Mapping**: Added compatibility handling for Codex model aliases such as `gpt-5.4-high`, preserving common request parameters during conversion.
  - **Codex 模型映射**：补充 `gpt-5.4-high` 等 Codex 模型别名兼容，并在转换时保留常用请求参数。
- **Streaming Output Shape**: Codex streaming output is normalized into Chat Completions chunk format for clients that expect OpenAI-compatible SSE.
  - **流式输出形态**：将 Codex 流式输出规范化为 Chat Completions chunk 格式，适配期望 OpenAI 兼容 SSE 的客户端。

## Breaking Changes / 不兼容更新

- **Attention**: No backend API or local data format breaking changes are expected in this stable release.
  - **必读**：本正式版预计没有后端 API 或本地数据格式不兼容更新。

## Refactor & Performance / 重构与优化

- **Proxy Conversion Cleanup**: Reduced duplicated Codex request and response conversion logic, making future compatibility fixes easier to maintain.
  - **代理转换清理**：减少 Codex 请求与响应转换中的重复逻辑，方便后续维护兼容性修复。
- **Frontend View Cleanup**: Simplified dashboard, help, release notes, and trend view code while keeping the visible behavior focused on the new workflows.
  - **前端视图清理**：简化仪表盘、使用说明、版本说明和趋势页面代码，同时保持新流程的展示体验。
- **Regression Coverage**: Added backend and frontend regression coverage for Codex routing/conversion, dashboard activity data, help content behavior, and usage trend rendering.
  - **回归覆盖**：补充 Codex 路由与转换、仪表盘活跃数据、使用说明行为和用量趋势渲染相关的后端与前端回归验证。
