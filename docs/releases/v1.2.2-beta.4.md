# v1.2.2-beta.4 - Model routing and quota guard / 模型路由与额度保护

This beta fixes model-aware gateway routing and restores quota protection when selected accounts are near the switch threshold. It also tightens the release workflow validation before publishing assets.

本次 beta 修复按模型识别网关后端和已选账号低额度时的切换保护，并加固发布流程中的构建前验证。

## Features / 新特性

- **Model-first gateway routing**: The gateway routing page now starts from the model, then shows which gateways can provide that model. Advanced per-client fallback chains are still available.
  - **模型优先路由**：网关路由页面现在先选模型，再展示可提供该模型的网关；高级区仍保留每个客户端入口的备用链配置。
- **Model-aware backend selection**: Claude, Codex, and OpenAI-compatible routes infer direct providers from model IDs, so `deepseek-*`, `mimo-*`, `kimi-*`, `glm-*`, and similar models are sent to the matching provider.
  - **按模型选择后端**：Claude、Codex 和 OpenAI 兼容入口会根据模型 ID 推导真实后端，`deepseek-*`、`mimo-*`、`kimi-*`、`glm-*` 等模型会转发到对应厂商。
- **Release workflow validation**: The Windows release job now installs frontend dependencies and runs backend/frontend tests before producing installer assets.
  - **发布流程验证**：Windows 发布任务现在会先安装前端依赖并运行后端、前端测试，再生成安装包附件。

## Bug Fixes / 问题修复

- **DeepSeek routing**: Claude requests using DeepSeek models no longer go to the Anthropic upstream and retry indefinitely.
  - **DeepSeek 路由**：Claude 请求选择 DeepSeek 模型时不再转发到 Anthropic 上游，避免持续重试。
- **DeepSeek model ID**: Legacy `deepseek-v4-pro[1m]` settings are normalized to the official `deepseek-v4-pro` model ID before forwarding upstream.
  - **DeepSeek 模型 ID**：旧配置中的 `deepseek-v4-pro[1m]` 会在转发前归一化为官方 `deepseek-v4-pro`。
- **Quota switching**: Selected accounts are now treated as a scheduling preference, not a way to bypass low-quota protection. A selected low-quota account is skipped while another active account for the same provider exists.
  - **额度切换**：已选账号现在是调度偏好，不会绕过低额度保护；同厂商还有 active 账号时，会跳过已选的 low 账号。
- **Config persistence**: Frontend config saving now preserves all top-level backend config fields, including provider-specific URL and Prem proxy settings.
  - **配置保存**：前端保存配置时会保留后端所有顶层配置字段，包括厂商 URL 和 Prem 代理设置。

## Tests / 测试

- Added routing regression tests for model-inferred providers and DeepSeek request normalization.
  - 新增按模型推导后端和 DeepSeek 请求归一化的路由回归测试。
- Added scheduler regression tests for selected low-quota accounts in queue, balanced, and preferred credential modes.
  - 新增队列、平衡和优先凭据模式下已选低额度账号的调度回归测试。
- Added persistence and recorder coverage for local JSON stores, client config helpers, and log sanitization.
  - 新增本地 JSON 存储、客户端配置工具和日志脱敏的测试覆盖。
