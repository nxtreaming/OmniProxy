# v1.2.3-beta.1 - Gateway operations and Codex workspace identities / 网关运维增强与 Codex 空间身份

This beta starts the `v1.2.3` line with operational tools for managing local OmniProxy setups, diagnosing gateway routes, and reviewing account health. It also fixes Codex `auth.json` imports for ChatGPT accounts that contain multiple workspace credentials under the same email.

本次 beta 开启 `v1.2.3` 版本线，重点增强本地 OmniProxy 配置管理、网关路由诊断和账号健康观察能力。同时修复同一个 ChatGPT 邮箱下存在多个空间凭证时，Codex `auth.json` 导入会互相覆盖的问题。

## Features / 新特性

- **Config snapshots**: Added local config snapshot export, import, restore, and delete flows for safer iterative setup changes.
  - **配置快照**：新增本地配置快照的导出、导入、恢复和删除流程，便于安全试错与回滚。
- **Provider model catalog sync**: Added provider model catalog synchronization and UI status so route setup can start from current backend model lists.
  - **厂商模型目录同步**：新增厂商模型目录同步和前端状态展示，路由配置可基于最新后端模型列表展开。
- **Gateway route diagnostics**: Added route diagnostics that explain matched provider, credential type, target URL, candidate accounts, and blocking reasons.
  - **网关路由诊断**：新增路由诊断，可展示匹配厂商、凭据类型、目标地址、候选账号和阻塞原因。
- **Route strategy presets**: Added model-route strategy presets to make common backend fallback chains faster to apply.
  - **路由策略预设**：新增模型路由策略预设，常见后端备用链可更快套用。
- **Diagnostics export**: Added a diagnostics bundle export path for troubleshooting app state, routes, and recent runtime signals.
  - **诊断导出**：新增诊断包导出，用于排查应用状态、路由配置和近期运行信号。

## Bug Fixes / 问题修复

- **Codex workspace identities**: Codex `auth.json` identity matching now uses `account_id` together with email, allowing the same ChatGPT email to store multiple workspace credentials.
  - **Codex 空间身份**：Codex `auth.json` 身份匹配现在同时使用邮箱和 `account_id`，同一 ChatGPT 邮箱下的多个空间凭证可以并存。
- **Codex auth sync**: Re-importing or one-click syncing the same Codex `account_id` updates the existing credential instead of creating duplicates, while different `account_id` values are preserved separately.
  - **Codex 凭据同步**：重复导入或一键同步相同 Codex `account_id` 会更新已有凭据；不同 `account_id` 会保留为独立账号。
- **Credential visibility**: The account list now surfaces Codex `account_id` so users can distinguish same-email workspace credentials without exposing token secrets.
  - **凭据可见性**：账号列表现在展示 Codex `account_id`，用户可区分同邮箱空间凭据，同时不暴露令牌内容。

## Refactor & Performance / 重构与优化

- **Proxy routing structure**: Split shared router and service behavior into smaller focused files, keeping routing, target resolution, and HTTP service handling easier to test.
  - **代理路由结构**：拆分共享路由与服务逻辑，让路由匹配、目标解析和 HTTP 服务处理更容易测试与维护。
- **Dashboard health signals**: Added account health scores, watch/risk groupings, active request visibility, and richer dashboard history summaries.
  - **仪表盘健康信号**：新增账号健康分、关注/风险分组、活跃请求可见性和更丰富的历史概览。
- **Quickstart previews**: Added client configuration previews before writing local tool config files.
  - **快速开始预览**：在写入本地工具配置前提供客户端配置预览。
- **History analysis**: Split history analytics into dedicated trend, rank, table, and model insight panels for a denser operations view.
  - **历史分析**：将历史分析拆分为趋势、排行、表格和模型洞察面板，提升运维视图的信息密度。
