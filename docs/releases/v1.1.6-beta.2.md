# v1.1.6-beta.2 - tooltip and navigation polish / 悬浮窗与导航交互优化

This beta continues the `v1.1.6` line with focused desktop UI fixes around hover details, trend charts, request history analytics, and page navigation. It keeps the production release profile while validating the interaction regressions found after `v1.1.6-beta.1`.

本次 beta 延续 `v1.1.6` 版本线，集中修复桌面端悬浮详情、趋势图、请求历史分析和页面跳转相关的交互问题。该版本继续使用常规发布 profile，用于验证 `v1.1.6-beta.1` 后发现的界面回归。

## Improvements / 优化

- **Unified hover details**: Dashboard activity calendar, usage trend charts, usage trend daily detail, and request-history daily usage now share the same custom tooltip visual language.
  - **统一悬浮详情**：仪表盘请求活跃日历、用量趋势图、用量趋势每日明细、请求历史每日用量统一使用同一套自定义悬浮窗视觉。
- **Trend chart interactions**: Usage trend points now expose larger hit areas, active guide lines, and richer token/request breakdowns on hover and keyboard focus.
  - **趋势图交互**：用量趋势点位加入更大的命中区域、激活辅助线，并在悬浮和键盘聚焦时展示更完整的 Token / 请求拆分。
- **Request history analytics**: The request-history daily usage chart now uses a custom tooltip instead of the browser native title tooltip, matching the rest of the app.
  - **请求历史分析**：请求历史每日用量图不再使用浏览器原生 title 提示，改为与其他页面一致的自定义悬浮窗。
- **Daily detail popover**: The usage trend daily table now opens a compact detail popover with viewport-aware positioning.
  - **每日明细弹窗**：用量趋势每日表格新增紧凑明细弹窗，并根据可视区域自动避让边缘。

## Bug Fixes / 问题修复

- **Dashboard heatmap tooltip**: Fixed the activity-calendar hover tooltip not appearing reliably in the desktop WebView.
  - **仪表盘热力图悬浮窗**：修复请求活跃日历悬浮窗在桌面 WebView 中不稳定显示的问题。
- **Popover placement**: Fixed daily detail popovers appearing in the bottom-right corner or being clipped by the content area.
  - **弹窗位置**：修复每日明细弹窗漂到右下角或被内容区域裁切的问题。
- **Page navigation**: Routed all in-app tab changes through the same navigation handler and paused scroll saving during page transitions to prevent failed-looking page jumps.
  - **页面跳转**：统一应用内页面切换入口，并在页面切换期间暂停滚动位置保存，避免跳转后看起来停留在错误位置。
- **Dark mode tooltip colors**: Fixed body-level teleported tooltips so they keep the correct dark-mode surface color.
  - **深色模式悬浮窗颜色**：修复 Teleport 到 body 后的悬浮窗在深色模式下背景色不一致的问题。

## Compatibility / 兼容性

- **Attention**: No backend API, proxy route, credential format, or local data format breaking changes are expected in this beta release.
  - **必读**：本 beta 预计没有后端 API、代理路由、凭据格式或本地数据格式不兼容更新。
