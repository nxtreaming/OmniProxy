# v1.2.2-beta.7 - Codex multi-model setup and automatic credentials / Codex 多模型配置与凭据自动匹配

This beta continues the `v1.2.2` model-routing line with focused fixes for Codex one-click setup and gateway credential selection. Codex can now write multiple model profiles from the one-click setup page, while gateway routes default to automatic credential matching and preserve in-progress selections while users navigate the app.

本次 beta 继续打磨 `v1.2.2` 的模型路由能力，重点修正 Codex 一键配置和网关凭据选择体验。Codex 现在可以从一键配置页写入多个模型配置；网关路由默认使用自动凭据匹配，并在用户切换页面时保留尚未保存的选择。

## Features / 新特性

- **Codex multi-model setup**: Codex one-click setup now supports selecting up to four models and writes separate generated profile files for the selected models.
  - **Codex 多模型配置**：Codex 一键配置现在最多可选择 4 个模型，并为所选模型写入独立的生成配置文件。
- **Codex default model set**: The default Codex selection is now GPT-5.5, GPT-5.4, and GPT-5.4 mini, with GPT-5.5 as the primary route model.
  - **Codex 默认模型组**：Codex 默认选择 GPT-5.5、GPT-5.4 和 GPT-5.4 mini，并以 GPT-5.5 作为主路由模型。
- **Generated Codex profiles**: Restore now removes generated OmniProxy Codex profile files so users can safely return to their previous local Codex config.
  - **生成的 Codex 配置清理**：恢复配置时会删除 OmniProxy 生成的 Codex 模型配置文件，便于安全回退到原本的本地 Codex 配置。

## UX / 体验优化

- **Automatic credential matching**: Gateway route credential selectors now default to automatic matching, allowing OmniProxy to choose an available credential type for the selected backend.
  - **凭据自动匹配**：网关路由凭据选择现在默认使用自动匹配，由 OmniProxy 根据后端可用凭据选择可用类型。
- **Preserved route edits**: Unsaved gateway route edits are preserved across page navigation and refreshes triggered by other app actions.
  - **保留未保存路由编辑**：用户在网关路由页做出的未保存修改，会在切换页面和其他操作触发刷新时保留。
- **Pure route reads**: Opening the model-routing page or switching selected models no longer creates hidden route entries until the user edits the route.
  - **路由读取无副作用**：打开模型路由页或切换模型不会再隐式创建路由，只有实际编辑时才写入草稿配置。

## Bug Fixes / 问题修复

- **Credential selection persistence**: Fixed selected credential types such as Codex `auth.json` reverting to API key after navigating away and back.
  - **凭据选择持久性**：修复选择 Codex `auth.json` 等凭据类型后，切换页面再回来又变回 API Key 的问题。
- **Codex automatic routing**: Codex OpenAI-compatible routes now prefer Codex `auth.json` when credential type is automatic, while still falling back to other usable OpenAI credentials.
  - **Codex 自动路由**：Codex 的 OpenAI 兼容路由在自动凭据模式下会优先使用 Codex `auth.json`，同时仍可回退到其他可用 OpenAI 凭据。
- **Model route mutation scope**: Model route display state now stays separate from mutation state, reducing accidental config changes while browsing route options.
  - **模型路由修改边界**：模型路由展示状态与写入状态分离，减少浏览路由选项时意外改动配置的情况。

## Tests / 测试

- Added backend coverage for Codex multi-model configuration, generated profile files, maximum model limits, and DeepSeek `[1m]` Codex normalization.
  - 新增 Codex 多模型配置、生成配置文件、最多模型数量和 DeepSeek `[1m]` Codex 归一化的后端测试。
- Updated config and proxy route coverage for automatic credential matching and Codex route defaults.
  - 更新配置与代理路由测试，覆盖自动凭据匹配和 Codex 默认路由行为。
