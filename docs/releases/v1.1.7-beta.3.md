# v1.1.7-beta.3 - focus mode automation / 放心刷自动化

This beta continues the `v1.1.7` line with a new "放心刷" workflow for local CLI-driven AI tasks. OmniProxy can detect supported CLI request activity, open a short-video or media app while the task is running, pause playback when the task becomes idle, and return focus to the original CLI window.

本次 beta 继续完善 `v1.1.7` 版本线，新增「放心刷」工作流。OmniProxy 可以检测受支持 CLI 的请求活动，在任务运行时打开短视频或媒体应用，任务空闲后自动暂停播放，并把焦点切回原 CLI 窗口。

## Features / 新特性

- **放心刷**: Added a configurable task automation section in global settings. It can watch Codex, Claude Code, Claude Desktop, OpenCode, Gemini CLI, DeepSeek-TUI, and Pi Coding Agent traffic that goes through OmniProxy.
  - **放心刷**：全局设置新增可配置的放心刷区域，可监听经过 OmniProxy 的 Codex、Claude Code、Claude Desktop、OpenCode、Gemini CLI、DeepSeek-TUI 和 Pi Coding Agent 请求活动。
- **Launch presets**: Added built-in presets for Douyin and Bilibili. Presets prefer Windows desktop apps or shortcuts and fall back to the official website when a local app is not found.
  - **打开预设**：新增抖音和哔哩哔哩预设。预设会优先打开 Windows 桌面端或快捷方式，找不到本地程序时回退到官网。
- **Custom launch target**: Users can still provide a custom executable, shortcut, or URL as the launch target.
  - **自定义目标**：仍支持填写自定义 exe、lnk 快捷方式或网址作为打开目标。

## Improvements / 优化

- **CLI focus restore**: Improved Windows foreground activation by restoring the captured CLI window, temporarily attaching input threads, toggling topmost state, and retrying foreground focus.
  - **CLI 焦点恢复**：增强 Windows 前台切换逻辑，会恢复记录到的 CLI 窗口、临时关联输入线程、短暂置顶再取消置顶，并重试前台聚焦。
- **Playback pause and resume**: OmniProxy now sends Space to pause the active playback window before returning to the CLI, remembers that window, and resumes it on the next task start.
  - **播放暂停与恢复**：任务结束切回 CLI 前会发送空格暂停当前播放窗口，并记录该窗口；下一轮任务开始时会切回该窗口并恢复播放。
- **Bilibili split-window support**: Bilibili desktop playback windows are handled by remembering the actual window paused by OmniProxy, so resume targets the playback window instead of the home feed window.
  - **哔哩哔哩分离窗口适配**：通过记录实际被 OmniProxy 暂停的窗口，下一轮恢复会对准播放窗口，而不是主页信息流窗口。
- **Return timing controls**: Added separate idle detection and return-wait timing controls so users can tune when a task is considered done and how long to wait before switching back.
  - **回切时序控制**：新增独立的空闲判定和回切前等待设置，便于调整任务结束判定和切回 CLI 的节奏。

## Compatibility / 兼容性

- **Attention**: This beta adds backward-compatible local configuration fields for "放心刷". Existing config files continue to load with defaults for the new options.
  - **必读**：本 beta 新增「放心刷」相关本地配置字段，属于向后兼容变更；旧配置会自动使用新选项默认值。
- **Attention**: Only CLI requests that pass through the local OmniProxy gateway can trigger the workflow.
  - **必读**：只有经过本机 OmniProxy 代理的 CLI 请求才会触发该工作流。
