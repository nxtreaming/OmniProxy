# v1.1.8-beta.7 - update scheduling polish / 更新检查调度优化

This beta continues the `v1.1.8` validation line with follow-up update UX fixes. The About page now uses the same no-extra-dialog install flow as the titlebar update popover, and OmniProxy will keep checking for available updates every four hours after startup.

本次 beta 继续验证 `v1.1.8` 版本线，补齐更新体验的后续修复。关于页现在与标题栏更新悬浮卡片使用同一套无额外弹窗安装流程，OmniProxy 启动后也会继续每 4 小时自动检查一次可用更新。

## Improvements / 优化

- **About page install flow**: Clicking `重启安装` from the About page now starts the installer directly, matching the titlebar update popover behavior.
  - **关于页安装流程统一**：在关于页点击 `重启安装` 会直接启动安装器，与标题栏更新悬浮卡片行为保持一致。
- **Periodic update checks**: OmniProxy still checks once shortly after startup, then automatically checks again every 4 hours while the app is running.
  - **周期更新检查**：OmniProxy 仍会在启动后短暂延迟检查一次，随后运行期间每 4 小时自动检查一次更新。
- **Timer cleanup**: The scheduled update check interval is cleared when the frontend app unmounts.
  - **定时器清理**：前端应用卸载时会清理周期更新检查定时器。

## Compatibility / 兼容性

- **Attention**: Update download, checksum verification, installer launch, release parsing, and manual update checks are unchanged.
  - **必读**：更新下载、SHA256 校验、安装器启动、发布版本解析和手动检查更新行为均不变。
