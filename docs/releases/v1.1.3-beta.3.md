# v1.1.3-beta.3 - Zo Computer gateway beta / Zo Computer 网关测试版

This beta adds Zo Computer as a local OmniProxy provider and expands one-click client configuration for coding agents.

本次 beta 新增 Zo Computer 本地网关，并补齐常用编码客户端的一键配置。

## Features / 新特性

- **Zo Computer Provider**: Added a Go-native Zo Computer gateway with OpenAI Chat Completions, OpenAI Responses, Anthropic Messages, and model-list compatibility endpoints.
  - **Zo Computer Provider**: 新增 Go 原生 Zo Computer 网关，支持 OpenAI Chat Completions、OpenAI Responses、Anthropic Messages 和模型列表兼容接口。
- **Coding Client Setup**: Added Zo Computer one-click configuration for Codex and Claude Code, including Opus 4.7 and Sonnet 4.6 model slots.
  - **编码客户端配置**: 新增 Codex 和 Claude Code 的 Zo Computer 一键配置，并写入 Opus 4.7 与 Sonnet 4.6 模型档位。
- **OpenCode and Pi Models**: Added Zo Computer model presets for OpenCode and Pi Coding Agent, including GPT-5.5, GPT-5.4, GLM 5, Gemini 3.1 Pro, MiniMax 2.7, and DeepSeek V4 Pro.
  - **OpenCode 和 Pi 模型**: 为 OpenCode 和 Pi Coding Agent 新增 Zo Computer 模型预设，包含 GPT-5.5、GPT-5.4、GLM 5、Gemini 3.1 Pro、MiniMax 2.7 和 DeepSeek V4 Pro。

## Fixes / 修复

- **Codex Responses Compatibility**: Fixed Codex requests that send zstd-compressed request bodies to the local Zo Responses endpoint.
  - **Codex Responses 兼容性**: 修复 Codex 向本地 Zo Responses 端点发送 zstd 压缩请求体时的 JSON 解析错误。
- **Responses Streaming Shape**: Improved synthetic OpenAI Responses streaming events so Codex can track active output items correctly.
  - **Responses 流式事件**: 优化合成的 OpenAI Responses 流式事件，确保 Codex 能正确跟踪输出项。
- **Zo Model Matching**: Improved Zo model alias matching for vendor-prefixed and label-based model names.
  - **Zo 模型匹配**: 优化 Zo 模型别名匹配，兼容带厂商前缀和显示标签的模型名称。
