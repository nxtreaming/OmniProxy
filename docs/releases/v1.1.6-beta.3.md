# v1.1.6-beta.3 - full history analytics / 完整历史统计修复

This beta continues the `v1.1.6` line with a focused request-history analytics fix. It keeps recent request rows lightweight while moving summary cards, trend charts, and ranking panels to database-backed aggregation so high-volume usage no longer hides earlier days.

本次 beta 延续 `v1.1.6` 版本线，集中修复请求历史统计问题。请求明细仍保持轻量加载，但顶部统计、趋势图和排行面板改为基于本地数据库聚合，避免高频请求时只看到最近几天数据。

## Improvements / 优化

- **Full-history analytics**: Request-history summary cards, daily usage charts, provider/tool/model rankings, and failure rankings now aggregate from the complete retained SQLite history instead of only the latest request rows.
  - **完整历史统计**：请求历史顶部统计、每日用量图、厂商 / 工具 / 模型排行和失败排行改为从完整保留的 SQLite 历史中聚合，不再只依赖最新请求明细。
- **Filtered summary refresh**: Request-history filters now refresh both the visible detail rows and the matching aggregate summary.
  - **筛选统计同步**：请求历史筛选条件会同时刷新可见明细和匹配的聚合统计。
- **Release bindings**: Wails bindings now expose the read-only request-history summary API for desktop and HTTP fallback modes.
  - **发布绑定同步**：Wails 绑定新增只读请求历史统计接口，桌面模式和 HTTP fallback 模式保持一致。

## Bug Fixes / 问题修复

- **Missing earlier days**: Fixed the request-history "daily usage · last 14 days" chart appearing empty for earlier dates when the newest 10,000 request rows were concentrated in the last few days.
  - **前几天数据缺失**：修复最新 10,000 条请求集中在最近几天时，「每日用量 · 近 14 天」图表前面日期看起来为空的问题。
- **Misleading totals**: Fixed request-history totals, failure rate, token count, and average duration being capped by the detail-row fetch limit.
  - **统计口径误导**：修复请求数、失败率、Token 和平均耗时受明细拉取上限影响的问题。

## Compatibility / 兼容性

- **Attention**: No proxy route, credential format, or local data format breaking changes are expected in this beta release. A new read-only local history summary endpoint is added for the desktop UI.
  - **必读**：本 beta 预计没有代理路由、凭据格式或本地数据格式不兼容更新。桌面端新增一个只读的本地历史统计接口。
