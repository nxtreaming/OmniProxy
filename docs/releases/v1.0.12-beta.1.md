# v1.0.12-beta.1 - Account selection beta / 账号选择调度 beta

This beta release focuses on account scheduling controls and the desktop quota experience. It replaces the old exclusive-account interaction with a selectable scheduling set, adds a coexisting local Dev executable build flow, and refreshes the README for the current provider and release workflow.
本次 beta 版本聚焦账号调度控制与桌面额度体验，将旧的单账号独占交互升级为可多选的调度集合，新增可与生产版共存的本地 Dev exe 构建流程，并更新 README 中的厂商和发布流程说明。

## Beta Notes / Beta 说明

- **Prerelease**: This build is intended for validating the account-selection scheduling workflow before a future stable release.
  - **预发布**: 此版本用于在后续正式版前验证账号选择调度工作流。
- **Release Installer**: GitHub Actions beta installers continue to use the normal release profile. The coexisting Dev executable is built locally with `scripts/build-dev.ps1`.
  - **发布安装包**: GitHub Actions 生成的 beta 安装包继续使用常规发布 profile。可共存的 Dev exe 通过 `scripts/build-dev.ps1` 本地构建。

## Features / 新特性

- **Selected Account Scheduling**: Providers now rotate all usable accounts by default, and switch to rotating only selected accounts once one or more accounts are selected.
  - **账号选择调度**: 厂商默认轮换全部可用账号；当选择一个或多个账号后，仅在已选账号内轮换。
- **Multi-account Selection**: Selecting one account behaves like the old only-use mode, while selecting multiple accounts allows routing through a controlled subset.
  - **多账号选择**: 选择单个账号等价于旧的仅使用模式；选择多个账号时，可在指定账号子集内轮换。
- **Selection API**: Added `PUT /api/tokens/{id}/selected` and matching Wails bindings for desktop selection state changes.
  - **选择接口**: 新增 `PUT /api/tokens/{id}/selected` 及对应 Wails 绑定，用于桌面端切换选择状态。
- **Coexisting Dev Build**: Added `scripts/build-dev.ps1` to build `OmniProxy-Dev.exe` with the dev profile, separate data directory, and separate local ports.
  - **共存 Dev 构建**: 新增 `scripts/build-dev.ps1`，可构建使用 dev profile、独立数据目录和独立本地端口的 `OmniProxy-Dev.exe`。

## Improvements / 优化

- **Quota Card Actions**: Quota cards now show account information first, then compact `选择 / 已选` and `刷新` actions using consistent status-sized controls.
  - **额度卡片操作**: 额度卡片先展示账号信息，再展示紧凑的 `选择 / 已选` 与 `刷新` 操作，并统一为状态标签大小。
- **Selected State Feedback**: The selected action now uses a persistent highlighted background and a distinct selected icon so selected accounts remain visually obvious.
  - **选中态反馈**: 已选操作使用持续高亮背景和独立选中图标，让账号是否参与选择集合更清晰。
- **Account Management Controls**: Enable and disable controls now live in Account Management, separating account availability from quota-page routing selection.
  - **账号管理控制**: 启用和停用操作归位到账号管理页，将账号可用性与额度页调度选择分离。
- **README Refresh**: Updated Chinese and English README content for supported providers, control APIs, Dev builds, and the current selection workflow.
  - **README 更新**: 更新中英文 README，补充当前支持厂商、控制 API、Dev 构建和账号选择调度流程。

## Bug Fixes / 问题修复

- **Right Window Resize Edge**: Added a Wails resize edge on the right side of the custom title bar shell so the desktop window can be resized from the right border again.
  - **右侧窗口缩放边缘**: 在自定义标题栏外壳右侧补充 Wails 缩放边缘，恢复从右边框调整窗口大小的能力。
- **Selection Persistence Semantics**: Selection no longer mutates other accounts' disabled state, so clearing a selection does not accidentally re-enable manually disabled accounts.
  - **选择状态语义**: 选择调度不再改写其他账号的停用状态，清空选择时不会误启用手动停用的账号。

## Breaking Changes / 不兼容更新

- **Attention**: No breaking changes are expected in this beta release. The previous `/exclusive` API remains available as a compatibility wrapper over the new selection model.
  - **必读**: 本 beta 版本预计没有不兼容更新。旧 `/exclusive` API 仍作为新选择模型的兼容包装保留。
