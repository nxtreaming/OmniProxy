# v1.1.1-beta.1 - sub2api adapter and Codex refresh fixes / sub2api 适配与 Codex 刷新修复

This beta release adds first-class sub2api account support for OpenAI/Codex, Anthropic, and Gemini-compatible routes, plus quota lookup through sub2api's usage endpoint. It also fixes Codex OAuth refresh behavior to match the Codex-compatible refresh flow used by sub2api.
本次 beta 版本新增 sub2api 账号适配，支持 OpenAI/Codex、Anthropic 和 Gemini 兼容路由，并通过 sub2api 用量接口读取额度；同时修复 Codex OAuth 刷新逻辑，使其与 sub2api 使用的 Codex 兼容刷新流程保持一致。

## Features / 新特性

- **sub2api Provider**: Added sub2api API Key accounts with per-account Base URL, so multiple sub2api services can coexist in the same account pool.
  - **sub2api Provider**: 新增 sub2api API Key 账号，并支持每个账号单独保存 Base URL，便于同时接入多个不同的 sub2api 服务。
- **sub2api Protocol Routes**: Added local routes for `/sub2api`, `/sub2api/anthropic`, and `/sub2api/gemini`, with protocol-specific authentication headers.
  - **sub2api 协议路由**: 新增 `/sub2api`、`/sub2api/anthropic` 和 `/sub2api/gemini` 本地入口，并按协议写入对应认证请求头。
- **Codex sub2api Setup**: Added one-click Codex configuration for the local sub2api route.
  - **Codex sub2api 配置**: 新增一键写入 Codex 使用本地 sub2api 入口的配置能力。
- **sub2api Quota Lookup**: Token validation now calls `/v1/usage` and parses quota, wallet balance, subscription limits, and rate limit windows.
  - **sub2api 额度查询**: 账号校验改为调用 `/v1/usage`，并解析总额度、钱包余额、订阅限额和速率窗口。

## Bug Fixes / 问题修复

- **Codex OAuth Refresh**: Refresh requests now include the Codex refresh scope and User-Agent, reuse stored `client_id` when present, and preserve/update nested expiration fields.
  - **Codex OAuth 刷新**: 刷新请求现在携带 Codex refresh scope 和 User-Agent，优先复用已保存的 `client_id`，并正确保留或更新嵌套过期时间字段。
- **sub2api Base URL Handling**: Versioned Base URLs avoid duplicate `/v1` path segments during validation and routing.
  - **sub2api Base URL 处理**: 带 `/v1` 的 Base URL 在校验和转发时不会重复拼接 `/v1`。

## Breaking Changes / 不兼容更新

- **Attention**: No backend API or local data format breaking changes are expected in this beta release.
  - **必读**: 本 beta 版本预计没有后端 API 或本地数据格式不兼容更新。

## Refactor & Performance / 重构与优化

- **Provider Plumbing**: Extended provider normalization, validation, routing, Wails bindings, and frontend forms to carry account-level Base URLs.
  - **Provider 管线**: 扩展 provider 归一化、校验、路由、Wails 绑定和前端表单，使其支持账号级 Base URL。
- **Regression Coverage**: Added backend tests for sub2api account storage, routing headers, usage parsing, and Codex refresh form fields.
  - **回归测试覆盖**: 补充 sub2api 账号保存、路由请求头、额度解析以及 Codex 刷新表单字段的后端测试。
