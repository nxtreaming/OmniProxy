# v1.1.7-beta.6 - Settings responsive polish and update popover / 设置页响应式与更新浮层

This beta continues the `v1.1.7` validation line with focused desktop UI polish for global settings and update discovery.

本次 beta 继续完善 `v1.1.7` 验证版本线，重点优化全局设置页的大屏响应式布局和标题栏更新提示。

## Improvements / 优化

- **Responsive global settings**: The global settings page now keeps a single page flow while letting each section adapt internally to wide screens.
  - **全局设置响应式**：设置页保持单列主流程，每个设置分区内部根据大屏宽度自适应排布。
- **Maintenance card density**: "应用维护" uses three cards in one row on large screens, with each card's action controls placed on the right.
  - **应用维护信息密度**：大屏下「应用维护」三个卡片同排展示，卡片操作放到右侧，减少空白。
- **Outbound provider grouping**: The outbound proxy provider selector lays out the three provider groups side by side on large screens.
  - **出站代理分组**：大屏下「走出站代理的接入厂商」三个分类并列展示。
- **Titlebar update popover**: The titlebar "新版本" hint now opens a lightweight Gemini-style non-modal popover near the titlebar button, without a full-screen dim background.
  - **标题栏更新浮层**：标题栏「新版本」提示改为 Gemini 风格非模态浮层，就近展开，不再出现全屏灰色遮罩。
- **Release behavior restored**: The titlebar update hint is shown only for non-development builds when a real update is detected.
  - **发布版行为恢复**：标题栏更新提示仅在非开发构建真实检测到新版本时显示。
