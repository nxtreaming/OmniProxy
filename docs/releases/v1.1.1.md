# v1.1.1 - sub2api support and account import polish / sub2api 支持与账号导入优化

This stable release consolidates the `v1.1.1` beta series. It adds first-class sub2api account support, improves Codex-compatible Responses forwarding, adds batch API Key import tools, and polishes API Key quota cards for dense account pools.
本正式版整合 `v1.1.1` beta 系列改动，新增 sub2api 账号适配，优化 Codex 兼容 Responses 转发，增加批量 API Key 导入工具，并改进密集账号池里的 API Key 额度卡片展示。

## Features / 新特性

- **sub2api Provider**: Added sub2api API Key accounts with per-account Base URL, so multiple sub2api services can coexist in the same account pool.
  - **sub2api Provider**: 新增 sub2api API Key 账号，并支持每个账号单独保存 Base URL，便于同时接入多个不同的 sub2api 服务。
- **sub2api Protocol Routes**: Added local routes for `/sub2api`, `/sub2api/anthropic`, and `/sub2api/gemini`, with protocol-specific authentication headers.
  - **sub2api 协议路由**: 新增 `/sub2api`、`/sub2api/anthropic` 和 `/sub2api/gemini` 本地入口，并按协议写入对应认证请求头。
- **Codex sub2api Setup**: Added one-click Codex configuration for the local sub2api route.
  - **Codex sub2api 配置**: 新增一键写入 Codex 使用本地 sub2api 入口的配置能力。
- **sub2api Quota Lookup**: Token validation now calls `/v1/usage` and parses quota, wallet balance, subscription limits, and rate limit windows.
  - **sub2api 额度查询**: 账号校验改为调用 `/v1/usage`，并解析总额度、钱包余额、订阅限额和速率窗口。
- **Batch API Key Import**: Added batch import for API Key accounts, including automatic account names and per-batch Base URL support for sub2api keys.
  - **批量 API Key 导入**: 新增 API Key 账号批量导入，支持自动账号命名，并支持为 sub2api Key 按批次填写 Base URL。
- **Codex Token Refresh Button**: OpenAI Codex `auth.json` accounts now show a manual refresh-token button.
  - **Codex 令牌刷新按钮**: OpenAI 的 Codex `auth.json` 账号新增手动刷新令牌按钮。

## Bug Fixes / 问题修复

- **Codex OAuth Refresh**: Refresh requests now include the Codex refresh scope and User-Agent, reuse stored `client_id` when present, and preserve/update nested expiration fields.
  - **Codex OAuth 刷新**: 刷新请求现在携带 Codex refresh scope 和 User-Agent，优先复用已保存的 `client_id`，并正确保留或更新嵌套过期时间字段。
- **sub2api Base URL Handling**: Versioned Base URLs avoid duplicate `/v1` path segments during validation and routing.
  - **sub2api Base URL 处理**: 带 `/v1` 的 Base URL 在校验和转发时不会重复拼接 `/v1`。
- **sub2api Codex Responses**: `/sub2api/responses` is normalized to upstream `/v1/responses`.
  - **sub2api Codex Responses**: `/sub2api/responses` 会归一化为上游 `/v1/responses` 路径。
- **sub2api Image Tool Guard**: Local sub2api OpenAI Responses requests strip Codex `image_generation` tools, related `tool_choice` fields, and image-generation include selectors before forwarding ordinary text requests.
  - **sub2api 图片工具防误判**: 本地 sub2api OpenAI Responses 请求会在转发普通文本请求前移除 Codex `image_generation` 工具、相关 `tool_choice` 字段和图片生成 include 项，避免被上游分组图片权限误拦截。
- **Quota Card Layout**: API Key quota cards now use a dedicated two-column metrics layout when no subscription quota windows are present.
  - **额度卡片布局**: API Key 账号在没有订阅额度窗口时，改用专门的双列指标布局。
- **Token Usage Readability**: Proxy token usage is split into separate `Token`, `入`, and `出` metrics instead of one squeezed inline string.
  - **Token 用量可读性**: 代理 Token 用量拆分为 `Token`、`入`、`出` 三个指标，避免被窄卡片挤压。

## Breaking Changes / 不兼容更新

- **Attention**: No backend API or local data format breaking changes are expected in this release. Existing one-click Codex configuration can be rerun to adopt the new local sub2api route.
  - **必读**: 本版本预计没有后端 API 或本地数据格式不兼容更新。已有 Codex 一键配置可重新执行，以写入新的本地 sub2api 入口。

## Refactor & Performance / 重构与优化

- **Provider Plumbing**: Extended provider normalization, validation, routing, Wails bindings, and frontend forms to carry account-level Base URLs.
  - **Provider 管线**: 扩展 provider 归一化、校验、路由、Wails 绑定和前端表单，使其支持账号级 Base URL。
- **Responsive Layout**: Added responsive handling for the new API-only quota card layout.
  - **响应式布局**: 为新的 API-only 额度卡片布局补充窄屏降级处理。
- **Regression Coverage**: Added backend and frontend tests for sub2api storage, routing, quota parsing, Codex refresh, image-tool stripping, and batch import control API paths.
  - **回归测试覆盖**: 补充 sub2api 保存、路由、额度解析、Codex 刷新、图片工具移除，以及批量导入控制 API 路径的后端和前端测试。
