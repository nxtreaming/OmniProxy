# v1.0.11-beta.2 - OpenRouter beta / OpenRouter 接入 beta

This beta release focuses on the new OpenRouter integration and the desktop experience around model discovery, direct chat testing, quota visibility, and dashboard density. It is intended for validating OpenRouter API Key workflows before a future stable release.
本次 beta 版本聚焦 OpenRouter 接入，以及围绕模型发现、直接对话体验、额度可视化和仪表盘信息密度的桌面体验优化。该版本用于在后续正式版前验证 OpenRouter API Key 工作流。

## Features / 新特性

- **OpenRouter Provider**: Added OpenRouter as a first-class provider with its own base URL, account type, proxy route, validation path, and generated Wails bindings.
  - **OpenRouter 提供商**: 新增 OpenRouter 独立厂商入口，支持独立 Base URL、账号类型、代理路由、验证路径和 Wails 绑定。
- **OpenRouter Models**: Fetch and cache OpenRouter model metadata, including model IDs, display names, context length, and pricing metadata.
  - **OpenRouter 模型**: 支持拉取并缓存 OpenRouter 模型元数据，包括模型 ID、展示名称、上下文长度和价格信息。
- **OpenRouter Chat Playground**: Added a dedicated OpenRouter chat page for selecting a model and testing conversations directly inside OmniProxy.
  - **OpenRouter 对话体验页**: 新增独立 OpenRouter 对话页面，可选择模型并直接在 OmniProxy 内测试对话。
- **Free Model Filtering**: Added quick filtering for free OpenRouter models and paginated model browsing in account management.
  - **免费模型筛选**: 支持一键筛选 OpenRouter 免费模型，并在账号管理中分页浏览模型列表。
- **OpenCode Integration**: OpenCode configuration now includes an OmniProxy OpenRouter provider and model list when available.
  - **OpenCode 集成**: OpenCode 配置会写入 OmniProxy OpenRouter provider，并在可用时同步模型列表。

## Improvements / 优化

- **Quota Visibility**: OpenRouter API Key quota is now shown on both the quota page and the OpenRouter chat page.
  - **额度可见性**: OpenRouter API Key 额度会同时显示在额度页和 OpenRouter 对话页。
- **Unlimited OpenRouter Keys**: OpenRouter keys without a configured spend limit now display as unlimited instead of looking empty or exhausted.
  - **不限额 Key 展示**: 未设置消费上限的 OpenRouter Key 会显示为“不限制”，不再表现为空额度或误判耗尽。
- **Chat History**: OpenRouter conversations are saved locally per model, and user-message headers no longer show model IDs.
  - **对话记录**: OpenRouter 对话按模型保存在本地，用户消息标题不再显示模型 ID。
- **Chat Interaction Polish**: Added a generation indicator, smoother local streaming output, better input focus styling, clearer send-button contrast, and more stable message layout.
  - **对话交互打磨**: 增加生成动画、更平滑的本地流式输出、优化输入框焦点样式、发送按钮对比度和消息布局稳定性。
- **Dashboard Density**: Dashboard quota and daily usage panels now use compact paging and summary trends so long account lists and large charts do not stretch the home page.
  - **仪表盘密度优化**: 首页额度和分天用量面板改为紧凑分页与摘要趋势，避免账号列表和大图表拉长首页。

## Bug Fixes / 问题修复

- **OpenRouter Quota Refresh**: Fixed refresh feedback when OpenRouter `/key` returns usage without `limit` or `limit_remaining`.
  - **OpenRouter 额度刷新**: 修复 OpenRouter `/key` 只返回 usage、没有 `limit` 或 `limit_remaining` 时刷新后看起来无变化的问题。
- **Refresh Accounting**: Batch quota refresh now counts failed validation responses correctly instead of treating every HTTP response as success.
  - **刷新结果统计**: 批量刷新额度会正确统计验证失败结果，不再把所有 HTTP 返回都当成成功。
- **Model Selection UX**: Fixed OpenRouter chat model selection and model-list loading states for large model lists.
  - **模型选择体验**: 修复 OpenRouter 对话页在大量模型场景下的选择与加载状态体验。

## Breaking Changes / 不兼容更新

- **Attention**: No breaking changes are expected in this beta release.
  - **必读**: 本 beta 版本预计没有不兼容更新。
