# v1.2.2-beta.6 - Official defaults, 1M DeepSeek, and routing UX / 官方默认模型、DeepSeek 1M 与路由体验

This beta refines the model-routing workflow introduced in `v1.2.2-beta.5`. Codex now defaults to GPT-5.5, Claude Code defaults to Anthropic's official `default` model alias, DeepSeek V4 Pro has separate standard and `[1m]` Claude Code entries, and the model routing page has a simpler model-first setup flow.

本次 beta 继续打磨 `v1.2.2-beta.5` 引入的模型路由能力。Codex 默认模型改为 GPT-5.5，Claude Code 默认使用 Anthropic 官方 `default` 模型别名，DeepSeek V4 Pro 拆分普通版与 `[1m]` Claude Code 版，模型路由页也改成更直接的模型优先配置流程。

## Features / 新特性

- **Codex official presets**: Added GPT-5.5 and GPT-5.4 mini to the Codex one-click model selector, with GPT-5.5 as the new Codex default route and setup model.
  - **Codex 官方预设**：Codex 一键配置新增 GPT-5.5 与 GPT-5.4 mini，并将 GPT-5.5 作为新的 Codex 默认路由和默认写入模型。
- **Claude official aliases**: Added Claude Code official `default`, `sonnet`, `opus`, and `haiku` model aliases, and changed the Claude default route to `default`.
  - **Claude 官方别名**：新增 Claude Code 官方 `default`、`sonnet`、`opus`、`haiku` 模型别名，并将 Claude 默认路由改为 `default`。
- **DeepSeek 1M model**: Added a distinct `deepseek-v4-pro[1m]` Claude Code model option so users can activate Claude Code 1M context while OmniProxy still normalizes the upstream DeepSeek request to `deepseek-v4-pro`.
  - **DeepSeek 1M 模型**：新增独立的 `deepseek-v4-pro[1m]` Claude Code 模型选项，用于激活 Claude Code 1M 上下文；OmniProxy 转发给 DeepSeek 上游时仍归一化为 `deepseek-v4-pro`。

## UX / 体验优化

- **Model routing flow**: Simplified the page to a single path: select or search a model, choose the primary backend, add ordered fallback backends, then save.
  - **模型路由流程**：页面操作改为单一路径：选择或搜索模型、选择主后端、按顺序添加备用后端，然后保存。
- **Search and custom models**: Combined model search and custom-model creation, so pressing Enter selects the only match or creates a new route only when there are no matches.
  - **搜索与自定义模型**：合并模型搜索和自定义模型创建；回车会优先选择唯一匹配项，只有没有匹配时才创建新路由。
- **Fallback backend selection**: Replaced fallback toggle chips with an add-from-select workflow that disables the current primary backend and already-added fallbacks.
  - **备用后端选择**：备用后端从点选 chip 改为下拉添加，当前主后端和已添加备用项会被禁用。

## Bug Fixes / 问题修复

- **DeepSeek model identity**: Model route keys now preserve `deepseek-v4-pro[1m]` as a separate client-side model instead of collapsing it into `deepseek-v4-pro`.
  - **DeepSeek 模型身份**：模型路由键现在会保留 `deepseek-v4-pro[1m]` 作为独立客户端模型，不再直接合并为 `deepseek-v4-pro`。
- **Claude 1M support flag**: Claude Desktop route generation now marks only explicit `[1m]` models as 1M-capable.
  - **Claude 1M 标记**：Claude Desktop 路由生成现在只对显式 `[1m]` 模型标记 1M 能力。
- **Generated bindings**: Synced Wails TypeScript model bindings after backend request model changes.
  - **生成绑定**：同步后端请求模型变化后的 Wails TypeScript 模型绑定。

## Tests / 测试

- Added backend coverage for official Claude aliases, DeepSeek `[1m]` preservation, and upstream DeepSeek model normalization.
  - 新增官方 Claude 别名、DeepSeek `[1m]` 保留和 DeepSeek 上游模型归一化的后端测试。
- Added config coverage proving model routes keep `deepseek-v4-pro[1m]` as an independent route key.
  - 新增配置测试，确认模型路由会把 `deepseek-v4-pro[1m]` 作为独立路由键保存。
