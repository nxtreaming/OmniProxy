# v1.1.1-beta.2 - sub2api Codex responses fix / sub2api Codex Responses 修复

This beta patch fixes Codex requests sent through OmniProxy's local sub2api adapter. It prevents ordinary text prompts from being rejected by sub2api group image-generation permissions when Codex includes the Responses `image_generation` tool declaration by default.
本次 beta 补丁修复 Codex 通过 OmniProxy 本地 sub2api 入口发送请求时的问题：当 Codex 默认携带 Responses `image_generation` 工具声明时，普通文本提示不再被 sub2api 的分组图片生成权限误拦截。

## Features / 新特性

- None / 无。

## Bug Fixes / 问题修复

- **sub2api Codex Responses**: `/sub2api/responses` is now normalized to the upstream `/v1/responses` path.
  - **sub2api Codex Responses**: `/sub2api/responses` 现在会归一化为上游 `/v1/responses` 路径。
- **sub2api Image Tool Guard**: For local sub2api Responses requests, OmniProxy strips Codex's automatic `image_generation` tool declaration before forwarding text requests upstream.
  - **sub2api 图片工具防误判**: 本地 sub2api Responses 请求转发前会移除 Codex 自动带上的 `image_generation` 工具声明，避免文本请求被误判为图片生成请求。

## Breaking Changes / 不兼容更新

- **Attention**: No backend API or local data format breaking changes are expected in this beta release.
  - **必读**: 本 beta 版本预计没有后端 API 或本地数据格式不兼容更新。

## Refactor & Performance / 重构与优化

- **Regression Coverage**: Added backend tests for versionless sub2api Responses routing and image tool stripping.
  - **回归测试覆盖**: 补充不带 `/v1` 的 sub2api Responses 路由和图片工具移除逻辑的后端测试。
