# v1.1.7-beta.7 - Linux.do browser profiles and request count summary / Linux.do 浏览器登录态与请求数汇总修复

This beta continues the `v1.1.7` validation line with the Linux.do browser-based focus-mode workflow and a request-history persistence fix for high-volume local usage.

本次 beta 继续完善 `v1.1.7` 验证版本线，加入 Linux.do 浏览器登录态复用能力，并修复高请求量场景下请求数统计被明细上限影响的问题。

## Features / 新特性

- **Linux.do browser mode for focus automation**: "放心刷" can open Linux.do in a real browser session instead of the short-video/media idle workflow.
  - **Linux.do 浏览器模式**：「放心刷」可选择打开 Linux.do 浏览器会话，不再需要走短视频/媒体空闲暂停流程。
- **Browser profile picker**: The desktop app can scan local Edge, Chrome, and Firefox profiles so users can pick an existing logged-in profile directly.
  - **浏览器用户数据选择**：桌面端可读取本机 Edge、Chrome、Firefox 用户配置，让用户直接选择已有登录态的浏览器用户数据目录。

## Bug Fixes / 问题修复

- **Request count no longer stops at the detail limit**: Daily request totals are now written to an independent SQLite summary table, so the 50,000-row request-detail cap only affects detailed rows, not aggregate counts.
  - **请求数不再卡在明细上限**：每日请求总数独立写入 SQLite 汇总表，50,000 条请求明细上限只影响明细列表，不再影响总请求数统计。
- **Clearing request details keeps daily totals**: "Clear request history" removes detailed request rows while retaining daily request summary and billable token summary data.
  - **清空明细保留每日总计**：「清空请求历史」只删除请求明细，已记录的每日请求汇总和账单 Token 汇总会继续保留。
- **Retained summary migration**: Existing SQLite request rows are backfilled into the new daily summary table on first launch after upgrade.
  - **已有数据迁移**：升级后首次启动会把现有 SQLite 请求记录回填到新的每日汇总表。

## Improvements / 优化

- **Settings copy clarified**: The retention setting now describes daily summaries as request-count plus final-token summaries, separate from full request details.
  - **设置文案明确**：保留设置说明改为每日汇总记录请求数和最终 Token 用量，和完整请求明细分开理解。
- **History refresh after clearing details**: The frontend now refreshes summary data after clearing request details instead of temporarily showing an empty summary.
  - **清空后刷新统计**：清空请求明细后前端会重新拉取汇总数据，不再临时把统计卡片置空。
