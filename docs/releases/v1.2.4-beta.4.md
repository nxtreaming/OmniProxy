# v1.2.4-beta.4 - Upstream error visibility / 上游错误可见性修复

This beta fixes misleading `no active token available` errors after Forge or another provider returns a retryable upstream response and no additional credential is available.

本次 beta 修复 Forge 或其他厂商返回可重试的上游响应、且没有额外凭证可切换时，客户端错误被误报为 `no active token available` 的问题。

## Bug Fixes / 问题修复

- **Preserved upstream errors**: When the last available credential receives a retryable upstream response, OmniProxy now returns the original status, headers, and response body to the client instead of replacing them with a generic proxy error.
  - **保留上游错误**：最后一个可用凭证收到可重试的上游响应时，OmniProxy 现在会向客户端返回原始状态码、响应头和响应正文，不再用通用代理错误覆盖。
- **Forge diagnostics**: Codex can now display the real Forge error response, making context-limit, capacity, and provider-side failures distinguishable.
  - **Forge 诊断**：Codex 现在可以显示 Forge 的真实错误响应，便于区分上下文限制、容量不足和厂商侧故障。
- **Sanitized request history**: The request history records a length-limited error summary while continuing to redact credentials and other sensitive values.
  - **脱敏请求历史**：请求历史会记录限制长度后的错误摘要，并继续脱敏凭证及其他敏感内容。
- **Retry compatibility**: Multi-credential switching and configured fallback-provider routing remain unchanged.
  - **重试兼容性**：多凭证切换和已配置的备用厂商路由行为保持不变。
