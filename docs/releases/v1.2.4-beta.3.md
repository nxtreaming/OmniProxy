# v1.2.4-beta.3 - Forge Codex routing fix / Forge Codex 路由修复

This beta fixes Forge backend selections being saved but ignored for Codex Responses requests in v1.2.4-beta.2.

本次 beta 修复 v1.2.4-beta.2 中 Forge 后端选择虽已保存、但 Codex Responses 请求仍继续使用 OpenAI 的问题。

## Bug Fixes / 问题修复

- **Codex Responses routing**: Forge model routes and Codex gateway routes now forward `/codex/v1/responses` requests to Forge's `/v1/responses` endpoint.
  - **Codex Responses 路由**：Forge 模型路由和 Codex 网关路由现在会将 `/codex/v1/responses` 请求转发到 Forge 的 `/v1/responses` 接口。
- **Backend selector consistency**: Forge is now available in the Codex backend selector, matching the provider saved by shared model routing.
  - **后端选择一致性**：Codex 后端选择器现在会显示 Forge，与共享模型路由保存的厂商保持一致。
- **Forge Responses authentication**: Added coverage confirming Forge Responses requests use Bearer authentication and clear inherited `x-api-key` headers.
  - **Forge Responses 鉴权**：新增测试确认 Forge Responses 请求使用 Bearer 鉴权，并清除继承的 `x-api-key` 请求头。
- **Protocol documentation**: Updated UI help and README content to include Forge's documented OpenAI Responses support.
  - **协议文档**：更新界面帮助和 README，补充 Forge 官方文档所述的 OpenAI Responses 支持。
