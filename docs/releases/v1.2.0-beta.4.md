# v1.2.0-beta.4 - Xiaomi MiMo Token Plan service centers / 小米 MiMo Token Plan 服务中心

This beta continues the `v1.2.0` validation line with a routing fix for Xiaomi MiMo Token Plan accounts. OmniProxy now represents all three documented Token Plan service centers instead of treating accounts as only China or overseas, so European AMS accounts can use their correct upstream endpoints.

本次 beta 继续验证 `v1.2.0` 版本线，修复 Xiaomi MiMo Token Plan 账号的路由区域处理。OmniProxy 现在按照文档支持三个 Token Plan 服务中心，不再仅将账号区分为国内或海外，因此欧洲 AMS 账号可以使用正确的上游入口。

## Changes / 变更

- **Three Token Plan service centers**: Added explicit `CN`, `SGP`, and `AMS` region support for Xiaomi MiMo Token Plan credentials.
  - **三个 Token Plan 服务中心**：Xiaomi MiMo Token Plan 凭据现在明确支持 `CN`、`SGP` 和 `AMS` 区域。
- **AMS upstream routing**: Added the Europe AMS OpenAI-compatible and Anthropic-compatible Base URLs and route Token Plan requests according to the selected account's service center.
  - **AMS 上游路由**：新增欧洲 AMS 的 OpenAI 兼容和 Anthropic 兼容 Base URL，并根据所选账号所属服务中心转发 Token Plan 请求。
- **Account and settings UI**: Account editing now offers China, Singapore SGP, and Europe AMS service-center choices; settings expose the AMS Base URLs and label SGP explicitly as Singapore.
  - **账号与设置界面**：账号编辑现在提供中国区、新加坡 SGP 和欧洲 AMS 服务中心选项；设置页提供 AMS Base URL，并将 SGP 明确标注为新加坡。
- **Desktop binding synchronization**: Synchronized the Wails configuration model with the added AMS endpoint fields.
  - **桌面绑定同步**：Wails 配置模型已同步新增的 AMS 入口字段。

## Compatibility / 兼容性

- **Existing accounts remain valid**: Existing China and SGP Token Plan accounts keep their current routing behavior; previously accepted overseas/global aliases continue to map to SGP.
  - **现有账号继续有效**：已有中国区和 SGP Token Plan 账号保持原有路由行为；此前已接受的 overseas/global 别名继续映射到 SGP。
- **Configuration addition is backward compatible**: Existing configuration files load with default AMS endpoints when the new fields are absent.
  - **配置增量向后兼容**：既有配置文件缺少新增字段时会自动使用默认 AMS 入口加载。
