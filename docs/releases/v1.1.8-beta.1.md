# v1.1.8-beta.1 - Update installer flow and beta channel / 更新安装流程与 Beta 通道

This beta starts the `v1.1.8` validation line with focused improvements to the Windows update experience. It adds a restart-to-install flow after verified downloads and introduces an opt-in setting for detecting GitHub Pre-release/Beta updates.

本次 beta 开启 `v1.1.8` 验证版本线，重点优化 Windows 软件更新体验。更新包下载并校验完成后，OmniProxy 会提示重启安装；同时新增可选设置，用于检测 GitHub Pre-release/Beta 版本更新。

## Features / 新特性

- **Restart-to-install updates**: After an update installer is downloaded and SHA256 verified, OmniProxy now prompts the user to restart and complete installation.
  - **重启完成更新**：更新安装包下载并通过 SHA256 校验后，OmniProxy 会提示用户重启并完成安装。
- **Automatic installer handoff**: On Windows, the app starts the verified installer in auto-update mode, exits cleanly, waits for installation, and relaunches OmniProxy after the installer finishes.
  - **自动交接安装器**：Windows 下会以自动更新模式启动已校验安装器，当前应用正常退出，安装完成后重新打开 OmniProxy。
- **Beta update channel switch**: A new settings option controls whether OmniProxy includes GitHub Pre-release/Beta releases in update checks.
  - **Beta 更新通道开关**：设置页新增选项，可控制更新检测是否纳入 GitHub Pre-release/Beta 版本。

## Improvements / 优化

- **Update UI clarity**: About page and titlebar update prompts now distinguish Beta updates and show "restart install" wording for downloaded installers.
  - **更新界面更清晰**：关于页和标题栏更新提示会区分 Beta 更新，并在安装包下载后使用「重启安装」相关文案。
- **Stable channel remains default**: Existing users continue to receive stable-release update prompts unless they explicitly enable Beta update checks.
  - **默认仍为正式通道**：现有用户默认只接收正式版更新提醒，只有显式开启后才会检测 Beta 版本。

## Compatibility / 兼容性

- **Attention**: No proxy route or credential format breaking changes are expected in this beta release.
  - **必读**：本 beta 版本预计没有代理路由或凭据格式不兼容更新。
- **Attention**: The Beta update setting is a backward-compatible local configuration field. Existing configs continue to load with Beta checks disabled by default.
  - **必读**：Beta 更新设置是向后兼容的本地配置字段。已有配置会继续加载，并默认关闭 Beta 检测。
