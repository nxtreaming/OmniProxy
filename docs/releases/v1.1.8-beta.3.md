# v1.1.8-beta.3 - new-api gateway support / new-api 网关接入

This beta continues the `v1.1.8` validation line with first-class new-api gateway support. OmniProxy can now manage new-api API keys, route OpenAI / Anthropic / Gemini-compatible traffic to a configured new-api upstream, refresh key quota through new-api's token usage endpoint, and configure Codex to use the local new-api entrypoint with `gpt-5.5` by default.

本次 beta 继续验证 `v1.1.8` 版本线，新增一等 new-api 网关支持。OmniProxy 现在可以管理 new-api API Key，将 OpenAI / Anthropic / Gemini 兼容请求转发到配置的 new-api 上游，通过 new-api 的 Token 用量接口刷新 Key 额度，并可一键把 Codex 切换到本地 new-api 入口，默认模型为 `gpt-5.5`。

## Features / 新特性

- **new-api provider**: Account management now includes a `new-api` provider with a default upstream Base URL of `http://127.0.0.1:3000`.
  - **new-api 厂商**：账号管理新增 `new-api` 厂商，默认上游 Base URL 为 `http://127.0.0.1:3000`。
- **OpenAI / Anthropic / Gemini routing**: Local routes `/newapi`, `/newapi/anthropic`, and `/newapi/gemini` map to the matching new-api-compatible upstream protocols.
  - **OpenAI / Anthropic / Gemini 路由**：新增本地 `/newapi`、`/newapi/anthropic`、`/newapi/gemini`，分别转发到 new-api 兼容协议入口。
- **Protocol-aware auth headers**: OmniProxy injects `Authorization: Bearer`, `x-api-key`, or `x-goog-api-key` according to the selected local new-api protocol route.
  - **按协议注入鉴权头**：OmniProxy 会根据 new-api 本地协议入口分别注入 `Authorization: Bearer`、`x-api-key` 或 `x-goog-api-key`。
- **new-api quota refresh**: Account validation reads `/api/usage/token/` and stores total, used, available, and unlimited quota state in the account quota view.
  - **new-api 额度刷新**：账号校验会读取 `/api/usage/token/`，并在额度视图记录总额度、已用额度、可用额度和不限额状态。
- **Codex new-api setup**: One-click Codex setup can switch Codex to `http://127.0.0.1:<proxyPort>/newapi` with `gpt-5.5` as the default model.
  - **Codex new-api 一键配置**：一键配置可将 Codex 切换到 `http://127.0.0.1:<proxyPort>/newapi`，默认模型为 `gpt-5.5`。

## Improvements / 优化

- **Frontend settings and quick reference**: Settings, account creation, batch import, endpoint reference, and Wails bindings now include new-api.
  - **前端设置与速查**：设置页、账号添加、批量导入、接口速查和 Wails 绑定均已加入 new-api。
- **Documentation**: README and English README now document new-api support and the new Codex configuration endpoint.
  - **文档更新**：中文和英文 README 已补充 new-api 支持和新的 Codex 配置接口。

## Compatibility / 兼容性

- **Attention**: Existing configurations continue to load. If `newapiBaseUrl` is missing, OmniProxy falls back to `http://127.0.0.1:3000`.
  - **必读**：已有配置可继续加载。缺少 `newapiBaseUrl` 时，OmniProxy 会回退到 `http://127.0.0.1:3000`。
- **Attention**: Existing providers and local routes are unchanged; new-api is additive.
  - **必读**：已有厂商和本地路由不变，new-api 为新增能力。
