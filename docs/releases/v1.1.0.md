# v1.1.0 - Client routing and quota polish / 客户端路由与额度体验优化

This stable release consolidates the `v1.1.0` beta series. It adds Pi Coding Agent single-entry routing, TokenRouter provider support, selectable Claude Code model slots, improved Codex Free quota handling, and a refreshed Chinese README.
本正式版整合 `v1.1.0` beta 系列改动，新增 Pi Coding Agent 单入口路由、TokenRouter provider、Claude Code 模型槽位选择，修复 Codex Free 额度展示，并重写中文 README。

## Features / 新特性

- **Pi Coding Agent Single Entry**: Added one-click Pi Coding Agent configuration through `/pi-router/v1`, with OmniProxy dispatching requests by model.
  - **Pi Coding Agent 单入口**: 新增 Pi Coding Agent 一键配置，统一使用 `/pi-router/v1`，由 OmniProxy 按模型分流。
- **TokenRouter Provider**: Added TokenRouter API Key management, `tr_` key validation, configurable Base URL, and local OpenAI-compatible route `/tokenrouter/v1`.
  - **TokenRouter Provider**: 新增 TokenRouter API Key 管理、`tr_` 前缀校验、可配置 Base URL，以及本地 OpenAI 兼容入口 `/tokenrouter/v1`。
- **TokenRouter Client Models**: Added TokenRouter auto models such as `auto:balance`, `auto:quality`, `auto:speed`, and `auto:cost` for Pi/OpenCode configuration.
  - **TokenRouter 客户端模型**: 为 Pi/OpenCode 本地配置新增 `auto:balance`、`auto:quality`、`auto:speed`、`auto:cost` 等 TokenRouter auto 模型。
- **Selectable Claude Models**: Added Claude Code model selection in one-click setup, supporting up to 4 selected DeepSeek, MiMo, Kimi, and GLM model slots.
  - **Claude 模型选择**: 一键配置新增 Claude Code 模型选择，支持最多 4 个 DeepSeek、MiMo、Kimi、GLM 模型槽位。
- **README Redesign**: Reworked the Chinese README with a clearer project pitch, badges, architecture diagram, support matrix, release channel notes, and security model.
  - **README 改版**: 重写中文 README，补充项目定位、徽章、架构图、支持矩阵、发布通道和安全模型。

## Bug Fixes / 问题修复

- **Pi MiMo Routing**: Fixed Pi exposing `mimo-v2.5-pro[1m]` on the OpenAI-compatible route, which could cause Xiaomi MiMo upstream `400` responses and local cooldowns.
  - **Pi MiMo 路由**: 修复 Pi 在 OpenAI 兼容入口暴露 `mimo-v2.5-pro[1m]`，可能导致 Xiaomi MiMo 上游返回 `400` 并触发本地账号冷却的问题。
- **Pi OpenAI Routing**: Fixed Pi GPT/OpenAI model requests accidentally using Codex `auth.json` credentials instead of OpenAI API keys.
  - **Pi OpenAI 路由**: 修复 Pi 的 GPT/OpenAI 模型请求可能误用 Codex `auth.json` 凭据，而不是 OpenAI API Key 的问题。
- **TokenRouter Log Labels**: TokenRouter direct requests are now labeled as TokenRouter in local request logs instead of generic API clients.
  - **TokenRouter 日志标识**: TokenRouter 直连请求现在会在本地请求日志中标记为 TokenRouter，而不是泛化为 API Client。
- **Claude Model Validation**: Backend validation rejects empty selections, unsupported model IDs, and more than 4 unique selected models.
  - **Claude 模型校验**: 后端会拒绝空选择、不支持的模型 ID，以及超过 4 个去重模型的请求。
- **Codex Free Quota**: Free-plan Codex quota windows are displayed as weekly quota, and empty zero-value quota windows are hidden.
  - **Codex Free 额度**: Codex Free 的额度窗口会按周额度显示，并隐藏空的 0 值额度窗口。

## Breaking Changes / 不兼容更新

- **Attention**: No backend API or data format breaking changes are expected in this release. Existing one-click client configuration can be rerun to adopt the newer Pi and Claude layouts.
  - **必读**: 本版本预计没有后端接口或数据格式不兼容更新。已有客户端一键配置可重新执行，以写入新的 Pi 和 Claude 配置结构。

## Refactor & Performance / 重构与优化

- **Pi Configuration Layout**: Removed separate Pi provider entries for Anthropic, Gemini, OpenRouter, and custom gateways in favor of one model-routed provider.
  - **Pi 配置结构**: 移除独立的 Anthropic、Gemini、OpenRouter、自定义网关 provider，改为单 provider 模型分流。
- **Claude Settings Writer**: Added reusable Claude model slot writing logic and generated Wails bindings for custom model configuration.
  - **Claude 配置写入**: 新增可复用的 Claude 模型槽位写入逻辑，并更新自定义模型配置的 Wails 绑定。
- **Quota Remaining Calculation**: Subscription quota status now uses an effective remaining calculation that can fall back from primary to secondary quota windows.
  - **额度剩余计算**: 订阅额度状态使用有效剩余额度计算，可在需要时从 primary 窗口回退到 secondary 窗口。
- **Regression Coverage**: Added tests for Pi routing, TokenRouter validation, Claude model selection, and Codex Free quota parsing.
  - **回归测试覆盖**: 补充 Pi 路由、TokenRouter 校验、Claude 模型选择和 Codex Free 额度解析测试。
