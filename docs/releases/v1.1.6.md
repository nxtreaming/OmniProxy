# v1.1.6 - history analytics and desktop interaction polish / 历史统计与桌面交互优化

This stable release collects the validated `v1.1.6` beta line into a production release. It improves dashboard card balance, unifies hover details across analytics views, fixes page navigation regressions, and moves request-history analytics to database-backed aggregation so high-volume usage no longer hides earlier retained data.

本正式版汇总已验证的 `v1.1.6` beta 版本线，优化仪表盘卡片平衡、统一分析视图悬浮详情、修复页面跳转回归，并将请求历史统计改为基于数据库聚合，避免高频请求时隐藏更早保留数据。

## Features / 新特性

- **Full-history request analytics**: Request-history summary cards, daily usage charts, provider/tool/model rankings, and failure rankings now aggregate from the complete retained SQLite history instead of only the latest request rows.
  - **完整请求历史统计**：请求历史顶部统计、每日用量图、厂商 / 工具 / 模型排行和失败排行改为从完整保留的 SQLite 历史中聚合，不再只依赖最新请求明细。
- **Unified hover details**: Dashboard activity calendar, usage trend charts, usage trend daily detail, and request-history daily usage now share the same custom tooltip visual language.
  - **统一悬浮详情**：仪表盘请求活跃日历、用量趋势图、用量趋势每日明细、请求历史每日用量统一使用同一套自定义悬浮窗视觉。
- **Filtered history summary refresh**: Request-history filters now refresh both visible detail rows and matching aggregate summaries.
  - **筛选统计同步**：请求历史筛选条件会同时刷新可见明细和匹配的聚合统计。

## Improvements / 优化

- **Dashboard card balance**: The coding-tool account usage card and quota overview cards now keep compact, visually aligned layouts when only a few rows are present.
  - **仪表盘卡片平衡**：编程工具账号占用卡片和额度概览卡片在数据较少时保持紧凑、对齐的布局。
- **Trend chart interactions**: Usage trend points now expose larger hit areas, active guide lines, and richer token/request breakdowns on hover and keyboard focus.
  - **趋势图交互**：用量趋势点位加入更大的命中区域、激活辅助线，并在悬浮和键盘聚焦时展示更完整的 Token / 请求拆分。
- **Daily detail popover**: The usage trend daily table now opens a compact detail popover with viewport-aware positioning.
  - **每日明细弹窗**：用量趋势每日表格新增紧凑明细弹窗，并根据可视区域自动避让边缘。
- **Desktop release bindings**: Wails bindings expose the read-only request-history summary API for both desktop and HTTP fallback modes.
  - **桌面发布绑定**：Wails 绑定新增只读请求历史统计接口，桌面模式和 HTTP fallback 模式保持一致。

## Bug Fixes / 问题修复

- **Missing earlier days**: Fixed the request-history "daily usage · last 14 days" chart appearing empty for earlier dates when the newest 10,000 request rows were concentrated in the last few days.
  - **前几天数据缺失**：修复最新 10,000 条请求集中在最近几天时，「每日用量 · 近 14 天」图表前面日期看起来为空的问题。
- **Misleading totals**: Fixed request-history totals, failure rate, token count, and average duration being capped by the detail-row fetch limit.
  - **统计口径误导**：修复请求数、失败率、Token 和平均耗时受明细拉取上限影响的问题。
- **Dashboard sparse rows**: Fixed single-row dashboard card layouts that could appear overly stretched or uneven.
  - **仪表盘稀疏行**：修复仪表盘只有一条记录时卡片内容被拉散或高度不均的问题。
- **Page navigation**: Routed in-app tab changes through the same navigation handler and paused scroll saving during page transitions to prevent failed-looking page jumps.
  - **页面跳转**：统一应用内页面切换入口，并在页面切换期间暂停滚动位置保存，避免跳转后看起来停留在错误位置。
- **Tooltip reliability**: Fixed dashboard heatmap tooltips not appearing reliably in the desktop WebView and normalized dark-mode tooltip colors for body-level teleports.
  - **悬浮窗可靠性**：修复桌面 WebView 中热力图悬浮窗不稳定显示的问题，并统一 Teleport 到 body 后的深色模式背景色。

## Compatibility / 兼容性

- **Attention**: No proxy route, credential format, or local data format breaking changes are expected in this stable release. A new read-only local history summary endpoint is added for the desktop UI.
  - **必读**：本正式版预计没有代理路由、凭据格式或本地数据格式不兼容更新。桌面端新增一个只读的本地历史统计接口。
