# v1.2.2-beta.5 - Client model selection and model backend chains / 客户端模型选择与模型后端链

This beta corrects the boundary between one-click client configuration and gateway routing. Client setup now controls which model a coding tool sends, while gateway routing controls the ordered backend chain for that model.

本次 beta 修正一键配置与网关路由的职责边界：客户端配置负责让编程工具发送哪个模型，网关路由负责为该模型配置有序后端链。

## Features / 新特性

- **Codex model selection**: The one-click Codex setup can now choose GPT, DeepSeek, MiMo, Kimi, GLM, MiniMax, or a custom model ID before writing the local Codex config.
  - **Codex 模型选择**：一键配置 Codex 时可以选择 GPT、DeepSeek、MiMo、Kimi、GLM、MiniMax，或直接填写自定义模型 ID。
- **Model backend chains**: Gateway routing now persists `modelRoutes`, so each model can define a primary backend and ordered fallback backends such as `DeepSeek API -> Prem`.
  - **模型后端链**：网关路由现在持久化 `modelRoutes`，每个模型都可以设置主后端和有序备用后端，例如 `DeepSeek 官方 API -> Prem`。
- **Custom model routes**: The gateway routing page can add arbitrary model IDs and infer a sensible default provider while still allowing manual backend and upstream model overrides.
  - **自定义模型路由**：网关路由页面可以添加任意模型 ID，并自动推断默认厂商，同时仍支持手动指定后端和上游模型 ID。

## Bug Fixes / 问题修复

- **Routing semantics**: Client entries are no longer treated as the primary routing dimension when a request includes a model. Matching model routes take priority across Codex, Claude, OpenCode, and compatible OpenAI entries.
  - **路由语义**：请求包含模型时，不再把客户端入口当作主要路由维度；Codex、Claude、OpenCode 和 OpenAI 兼容入口都会优先命中模型路由。
- **Fallback model rewrite**: Explicit fallback model IDs are now applied to the upstream request body, not only recorded in logs and history.
  - **备用模型改写**：显式配置的备用模型 ID 现在会真正写入上游请求体，而不只是记录到日志和历史。
- **Codex flexibility**: Codex one-click setup no longer rejects non-preset model IDs, so local custom gateways and provider-specific model names can be configured directly.
  - **Codex 灵活性**：Codex 一键配置不再拒绝非预设模型 ID，可直接配置本地自定义网关或厂商专用模型名。
