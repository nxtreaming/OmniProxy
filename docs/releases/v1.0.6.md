# v1.0.6 - Xiaomi MiMo quota and fallback routing / 小米 MiMo 额度与兜底路由

This release adds Xiaomi MiMo Token Plan and pay-as-you-go balance visibility, improves MiMo routing fallback, and fixes dashboard/log layout issues.
本次版本新增小米 MiMo Token Plan 与按量 API 余额展示，改进 MiMo 路由兜底切换，并修复仪表盘和日志页面布局问题。

## Features / 新特性
- **Xiaomi MiMo**: Added Token Plan usage lookup and pay-as-you-go API balance lookup using the Xiaomi platform console session. @mibgb65-cloud
  - 新增小米 MiMo Token Plan 用量查询和按量 API 余额查询，使用小米平台控制台登录态读取额度信息。
- **Settings**: Added Xiaomi platform Cookie configuration and HAR import support for MiMo quota queries. @mibgb65-cloud
  - 新增小米平台 Cookie 配置和 HAR 导入能力，用于 MiMo 额度查询。
- **Dashboard**: Split subscription quota and API/balance status into separate overview sections. @mibgb65-cloud
  - 仪表盘额度概览拆分为订阅额度和 API / 余额状态两块展示。

## Bug Fixes / 问题修复
- **Xiaomi MiMo**: Fixed MiMo credential priority so Token Plan and pay-as-you-go API are fallback priorities, not exclusive modes. @mibgb65-cloud
  - 修复 MiMo 凭据优先级逻辑，使 Token Plan 与按量 API 互为兜底优先级，而不是互斥模式。
- **Xiaomi MiMo**: Added fallback switching after MiMo upstream credential/quota responses so Claude can continue with the alternate credential type. @mibgb65-cloud
  - MiMo 上游返回凭据或额度类响应后自动切换到另一类凭据，避免 Claude 直接失败。
- **Quota UI**: Removed subscription percentage displays from ordinary API keys and kept API keys on balance/status displays only. @mibgb65-cloud
  - 普通 API Key 不再显示订阅百分比，仅展示余额或 API 状态。
- **Logs UI**: Fixed real-time log rows overflowing the panel on narrower windows. @mibgb65-cloud
  - 修复实时日志页面在较窄窗口下日志行撑出面板的问题。

## Breaking Changes / 不兼容更新
- **Attention**: No breaking changes are expected in this release.
  - **必读**: 本版本预计没有不兼容更新。

## Refactor & Performance / 重构与优化
- **Quota Refresh**: Extended usage metadata plumbing so providers can expose balance status alongside subscription quota. @mibgb65-cloud
  - 扩展额度元数据链路，使供应商可以同时展示订阅额度和余额状态。
- **Security**: Ignored HAR capture files to reduce the risk of accidentally committing browser session data. @mibgb65-cloud
  - 忽略 HAR 抓包文件，降低误提交浏览器登录态数据的风险。
