# v1.1.2-bate.1 - Codex Chat Completions compatibility / Codex Chat Completions 兼容

This bate pre-release adds a Codex-authenticated OpenAI Chat Completions compatibility route. Clients can call `/codex/v1/chat/completions` with an OpenAI `auth.json` account, while OmniProxy converts the request to the Codex Responses backend and converts the result back to Chat Completions JSON or SSE chunks.
本次 bate 预发布新增基于 Codex `auth.json` 账号的 OpenAI Chat Completions 兼容入口。客户端可以调用 `/codex/v1/chat/completions`，由 OmniProxy 转换为 Codex Responses 后端请求，并把结果转换回 Chat Completions JSON 或 SSE chunk。

## Features / 新特性

- **Codex Chat Completions Route**: Added `/codex/v1/chat/completions` for clients that expect OpenAI Chat Completions while using Codex `auth.json` credentials.
  - **Codex Chat Completions 入口**: 新增 `/codex/v1/chat/completions`，让需要 OpenAI Chat Completions 协议的客户端也能使用 Codex `auth.json` 凭据。
- **Responses Request Conversion**: Chat messages, system/developer instructions, max token fields, reasoning effort, tools, and function calling metadata are converted into Codex Responses payloads.
  - **Responses 请求转换**: 将 Chat messages、system/developer instructions、max token 字段、reasoning effort、tools 与 function calling 元数据转换为 Codex Responses 请求体。
- **Chat Response Conversion**: Codex Responses JSON and SSE events are converted back into `chat.completion` or `chat.completion.chunk` responses.
  - **Chat 响应转换**: 将 Codex Responses JSON 与 SSE 事件转换回 `chat.completion` 或 `chat.completion.chunk` 响应。

## Bug Fixes / 问题修复

- **Codex Model Aliases**: Codex CLI model variants such as `gpt-5.4-high` are normalized before forwarding to the Codex Responses backend.
  - **Codex 模型别名**: `gpt-5.4-high` 等 Codex CLI 模型变体会在转发到 Codex Responses 后端前归一化。

## Breaking Changes / 不兼容更新

- **Attention**: No backend API or local data format breaking changes are expected in this bate release.
  - **必读**: 本 bate 版本预计没有后端 API 或本地数据格式不兼容更新。

## Refactor & Performance / 重构与优化

- **Regression Coverage**: Added backend tests for Codex prefix routing, request conversion, non-streaming Chat Completions output, and streaming SSE compatibility.
  - **回归测试覆盖**: 补充 Codex 前缀路由、请求转换、非流式 Chat Completions 输出和流式 SSE 兼容性的后端测试。
