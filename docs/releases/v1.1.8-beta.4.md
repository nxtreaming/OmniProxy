# v1.1.8-beta.4 - update flow and About page polish / 更新流程与关于页优化

This beta continues the `v1.1.8` validation line with a smoother in-app update flow. When OmniProxy finds a downloadable release package, it now starts the download in the background and only prompts after the verified package is ready, asking the user to restart and complete the update.

本次 beta 继续验证 `v1.1.8` 版本线，优化应用内更新体验。OmniProxy 发现可下载的发布安装包后，会直接在后台下载，安装包下载并校验完成后再提示用户重启完成更新。

## Improvements / 优化

- **Automatic update download**: Available updates with installer and checksum assets now start downloading automatically, without the extra confirmation dialog before download.
  - **自动下载更新**：当新版本包含安装包和校验文件时，会自动开始下载，不再先弹出下载确认框。
- **Ready-to-restart prompt**: After download and SHA256 verification, the prompt now says the new version is ready and asks the user to restart OmniProxy to complete the update.
  - **重启完成更新提示**：安装包下载并通过 SHA256 校验后，会提示“新版本已准备好”，引导用户重启完成更新。
- **About page update states**: The About page now shows clearer update states for downloading, ready, failed, and installing.
  - **关于页更新状态**：关于页现在更清晰地展示下载中、已准备好、下载失败和安装器已启动等状态。
- **Dark-mode update dialogs**: Element Plus update confirmation dialogs now match the dark desktop UI instead of opening as a bright white modal.
  - **深色模式弹窗适配**：更新确认弹窗已适配深色桌面 UI，不再显示突兀的白色弹窗。

## Compatibility / 兼容性

- **Attention**: The update package download and verification still use the existing Windows installer assets and SHA256 checksum assets.
  - **必读**：更新包下载与校验仍使用现有 Windows 安装包和 SHA256 校验文件。
- **Attention**: Manual update checks still work, but a detected downloadable update now enters the same automatic background download flow.
  - **必读**：手动检查更新仍可使用，但检测到可下载更新后也会进入自动后台下载流程。
