# v1.1.7-beta.8 - Linux.do focus automation fixes / Linux.do 放心刷窗口行为修复

This beta continues the `v1.1.7` validation line with focused fixes for the "放心刷" automation workflow. It makes Linux.do browser mode open the intended page reliably, prevents repeated browser launches across task rounds, and preserves existing browser and CLI window size when returning focus.

本次 beta 继续完善 `v1.1.7` 验证版本线，集中修复「放心刷」自动化工作流。Linux.do 浏览器模式现在会稳定打开目标网页，跨任务轮次不会重复启动浏览器，并且在浏览器与 CLI 之间切换焦点时不会改变已有窗口大小。

## Bug Fixes / 问题修复

- **Linux.do opens the real site**: The Linux.do preset now resolves to `https://linux.do/` instead of accidentally passing `preset:linuxdo` to the browser.
  - **Linux.do 正确跳转**：Linux.do 预设会解析为 `https://linux.do/`，不再把 `preset:linuxdo` 误传给浏览器。
- **No repeated Linux.do browser launches**: Linux.do mode remembers the opened browser session and focuses the existing browser window in later task rounds instead of opening another browser.
  - **不重复打开浏览器**：Linux.do 模式会记住已打开的浏览器会话，后续任务轮次优先切回已有浏览器窗口，不再重复启动新浏览器。
- **Browser comes to front after launch**: Linux.do browser mode now retries foreground activation for the selected browser window after opening the site.
  - **浏览器自动置前**：Linux.do 浏览器模式打开网页后会重试激活对应浏览器窗口，让窗口自动切到最前。
- **Maximized windows stay maximized**: Browser and CLI focus switching no longer uses unconditional `SW_RESTORE`, so maximized windows are not shrunk when OmniProxy brings them to the foreground.
  - **最大化窗口不再缩小**：浏览器和 CLI 焦点切换不再无条件调用 `SW_RESTORE`，最大化窗口被切到前台时不会被还原成普通大小。
- **Media targets preserve window size**: Douyin, Bilibili, and fallback URL launches now use `SW_SHOW` rather than `SW_SHOWNORMAL`, reducing the chance that existing media/browser windows are resized during automation.
  - **媒体目标保留窗口尺寸**：抖音、哔哩哔哩和备用网页打开时改用 `SW_SHOW`，降低已有媒体或浏览器窗口在自动化过程中被调整大小的概率。

## Improvements / 优化

- **Closed Linux.do windows can recover**: If the previously captured Linux.do browser window was closed, the next matching task round clears the stale session and opens Linux.do again.
  - **已关闭窗口可恢复**：如果上次记录的 Linux.do 浏览器窗口已关闭，下次匹配任务会清理旧会话并重新打开 Linux.do。
- **Default Chromium profile handling**: Edge and Chrome launches skip redundant `--user-data-dir` when the selected profile uses the browser's default user data root, while preserving explicitly configured custom user data directories.
  - **默认 Chromium 资料处理**：Edge 和 Chrome 使用默认用户数据根目录时不再重复传入 `--user-data-dir`，但仍保留用户显式配置的自定义用户数据目录。
